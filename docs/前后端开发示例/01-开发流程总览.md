# 模块开发流程总览

> BladeX 框架下前后端分离模块的标准开发流程（7 阶段模型）

## 各阶段详细说明

### 阶段 1: 开发前分析 & 原型设计

**目标**：明确模块需求和页面设计

**主要工作**：
1. 确定模块基本信息（名称、代码、路由、所属模块）
2. 设计原型页面（列表、表单、查询条件）
3. 确认技术栈和开发方向

**输出物**：
- 模块基本信息表
- 页面原型图/设计稿
- 字段清单

---

### 阶段 2: 数据库设计

**目标**：设计符合 BladeX 规范的表结构

**主要工作**：
1. 设计表结构（必须包含 BladeX 规范字段）
2. 生成建表 SQL 脚本
3. 执行建表 SQL

**输出物**：
- 建表 SQL 脚本
- 数据库表结构文档

---

### 阶段 3: 字典与权限配置分析

**目标**：完成系统配置，为开发做准备

**主要工作**：
1. 判断是否需要字典配置
2. 生成字典 SQL（系统字典存 blade_dict）
3. 设计菜单结构，生成权限 SQL（存 blade_menu）
4. 执行配置 SQL

**输出物**：
- 字典 SQL 脚本
- 菜单权限 SQL 脚本
- 权限码对照表

---

### 阶段 4: 接口契约定义

**目标**：定义前后端接口规范，支持并行开发

**主要工作**：
1. 定义接口列表（URL、方法、参数、响应）
2. 前后端确认接口契约
3. 后端编写 Controller 接口框架
4. 前端根据接口配置 Avue column

**输出物**：
- 接口清单文档
- 前后端接口确认书

---

### 阶段 5A: 后端开发

**目标**：完成后端代码开发

**主要工作**：
1. Entity 实体类（继承 TenantEntity）
2. VO 视图对象（继承 Entity）
3. Mapper 接口（继承 BaseMapper）
4. Service 接口（继承 BaseService）
5. ServiceImpl 实现（继承 BaseServiceImpl）
6. Wrapper 数据包装类（处理字典转换）
7. Controller 控制器（权限注解、接口实现）

**输出物**：
- 完整的后端代码
- 单元测试通过

---

### 阶段 5B: 前端开发

**目标**：完成前端页面开发

**主要工作**：
1. API 接口封装（axios 调用）
2. Vue 页面组件（Avue CRUD 配置）
3. 字典下拉配置
4. 权限按钮控制

**输出物**：
- API 文件
- Vue 页面组件

---

### 阶段 6: 联调测试

**目标**：验证前后端功能完整性

**主要工作**：
1. 执行所有 SQL 脚本（建表 + 字典 + 菜单）
2. 刷新 Redis 缓存
3. 后端接口测试（Swagger）
4. 前端功能测试
5. 权限测试
6. 问题修复

**输出物**：
- 测试报告
- 问题修复清单

---

### 阶段 7: 上线部署

**目标**：完成生产环境部署

**主要工作**：
1. SQL 脚本归档
2. 代码合并到主分支
3. 后端 Maven 打包
4. 前端生产构建
5. 生产环境部署
6. 部署验证

**输出物**：
- 部署文档
- 上线报告

---
