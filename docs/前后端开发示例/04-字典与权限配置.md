# 字典与权限配置指南

> BladeX 框架下的字典配置和菜单权限配置方法

---

## 一、字典配置

### 1.1 字典配置决策树

```
是否需要下拉选择？
    │
    ├── NO → 跳过字典配置
    │
    └── YES → 继续分析
            │
            ├── 系统字典（推荐）
            │   - 可复用的字典（如：消息类型、订单状态）
            │   - 存储在 blade_dict 表
            │   - 前端通过 /blade-system/dict/dictionary 获取
            │
            └── 业务字典
                - 仅当前模块使用
                - 可存储在前端配置或独立字典表
```

### 1.2 blade_dict 表结构

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `id` | 主键（雪花算法） | 100001 |
| `parent_id` | 父主键（顶级为 0，子级指向父级 id） | 0 / 100001 |
| `code` | 字典编码（模块唯一标识） | 'message_type' |
| `dict_key` | 字典键值（**顶级固定为 -1**，子级为业务值） | -1 / 1 / 2 / 3 |
| `dict_value` | 字典显示名称 | '消息类型' / '通知' |
| `sort` | 排序 | 1, 2, 3... |
| `is_sealed` | 是否封存（0: 否，1: 是） | 0 |
| `status` | 状态（默认 1） | 1 |
| `is_deleted` | 是否删除（0: 否，1: 是） | 0 |

> **注意**：`blade_dict` 表无 `tenant_id`、`create_time` 字段

### 1.3 字典 SQL 脚本示例

**文件位置**: `ed-station-boot/sql/02_dict/dict_xxx.sql`

```sql
-- =============================================
-- 字典：消息类型
-- 字典编码：message_type
-- 所属模块：内部消息
-- =============================================

-- 1. 插入字典类型（父级，dict_key = -1）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES (100001, 0, 'message_type', '-1', '消息类型', 1, NULL, 0, 1, 0);

-- 2. 插入字典值（子级，dict_key 为实际业务值）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES 
(100002, 100001, 'message_type', '1', '通知', 1, NULL, 0, 1, 0),
(100003, 100001, 'message_type', '2', '公告', 2, NULL, 0, 1, 0),
(100004, 100001, 'message_type', '3', '待办', 3, NULL, 0, 1, 0);
```

### 1.4 参考现有字典数据

| 字典编码 | 父级 dict_key | 子级 dict_key | 说明 |
|---------|--------------|--------------|------|
| `sex` | '-1' | '1'(男), '2'(女) | 性别字典 |
| `notice` | '-1' | '1'~'6' | 通知类型 |
| `menu_type` | '-1' | '1', '2' | 菜单类型 |

### 1.5 后端字典枚举配置

**文件**: `src/main/java/org/springblade/common/enums/DictEnum.java`

```java

    // ... 其他字典
    
    /**
     * 消息类型（内部消息模块）
     */
    MESSAGE_TYPE("message_type"),
    
    // ... 其他字典
}
```

## 二、菜单权限配置

### 2.1 blade_menu 表结构

| 字段名 | 说明 | 示例 | 注意事项 |
|--------|------|------|---------|
| `id` | 主键（**雪花算法**） | 200001 | 无自增，需手动生成 |
| `parent_id` | 父主键 | 0 / 1123598815738675201 | 顶级为 0，子级指向父级 ID |
| `code` | 菜单编号（权限码） | 'innermessage' | 与 `@PreAuth(menu = "xxx")` 对应 |
| `name` | 菜单名称 | '内部消息' | 前端显示名称 |
| `alias` | 菜单别名 | 'menu' / 'add' / 'edit' | 菜单/按钮类型标识 |
| `path` | 请求地址 | '/desk/innermessage' | 前端路由或 API 地址 |
| `source` | 菜单资源（图标） | 'iconfont iconicon_sms' | 一级菜单用图标，按钮用按钮图标 |
| `sort` | 排序 | 1, 2, 3... | 同级菜单排序 |
| `category` | 菜单类型 | 1=一级菜单，2=按钮菜单 | - |
| `action` | 操作按钮类型 | 1=工具栏，2=操作栏，3=工具操作栏 | 仅按钮菜单使用 |
| `is_open` | 是否打开新页面 | 1 | 1=是，0=否 |
| `component` | 组件地址 | '' | 前端组件路径（可选） |
| `remark` | 备注 | NULL | - |
| `is_deleted` | 是否删除 | 0 | 0=否，1=是 |

> **注意**：无 `create_time`、`menu_name`、`menu_code`、`menu_uri`、`menu_type`、`menu_icon`、`component_path`、`is_inner`、`is_datascope`、`permission` 字段

### 2.2 菜单层级关系

```
顶级菜单 (parent_id = 0)
  └── 一级菜单 (parent_id = 顶级菜单 ID)
      └── 按钮/功能菜单 (parent_id = 一级菜单 ID)
```

### 2.3 所属模块查找

- 如果模块有父级（如 `desk`），需要查找对应父级 ID
- 参考现有数据：
  - `desk` 工作台 ID = `1123598815738675201`
  - `notice` 通知公告 ID = `1123598815738675202`

### 2.4 菜单 SQL 脚本示例

**文件位置**: `ed-station-boot/sql/03_menu/menu_xxx.sql`

```sql
-- =============================================
-- 菜单：内部消息
-- 所属模块：desk（工作台）
-- 父级菜单 ID: 1123598815738675201 (desk)
-- 创建时间：2026-04-02
-- =============================================

-- 1. 插入一级菜单（内部消息）
-- ID 使用雪花算法生成，parent_id 指向 desk 菜单 ID
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, source, sort, category, action, is_open, component, remark, is_deleted)
VALUES 
(200001, 1123598815738675201, 'innermessage', '内部消息', 'menu', '/desk/innermessage', 'iconfont iconicon_sms', 3, 1, 0, 1, '', NULL, 0);

-- 2. 插入按钮菜单（新增、修改、删除、查看）
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, source, sort, category, action, is_open, component, remark, is_deleted)
VALUES 
-- 新增按钮 (category=2, action=1)
(200002, 200001, 'innermessage_add', '新增', 'add', '/desk/innermessage/add', 'plus', 1, 2, 1, 1, '', NULL, 0),

-- 修改按钮 (category=2, action=2)
(200003, 200001, 'innermessage_edit', '修改', 'edit', '/desk/innermessage/edit', 'form', 2, 2, 2, 1, '', NULL, 0),

-- 删除按钮 (category=2, action=3)
(200004, 200001, 'innermessage_delete', '删除', 'delete', '/api/blade-desk/innermessage/remove', 'delete', 3, 2, 3, 1, '', NULL, 0),

-- 查看按钮 (category=2, action=2)
(200005, 200001, 'innermessage_view', '查看', 'view', '/desk/innermessage/view', 'file-text', 4, 2, 2, 1, '', NULL, 0);
```

### 2.5 参考现有菜单数据

| 菜单 code | id | parent_id | category | action | 说明 |
|----------|----|-----------|----------|--------|------|
| `desk` | 1123598815738675201 | 0 | 1 | 0 | 工作台（顶级） |
| `notice` | 1123598815738675202 | 1123598815738675201 | 1 | 0 | 通知公告 |
| `notice_add` | 1123598815738675219 | 1123598815738675202 | 2 | 1 | 新增按钮 |

## 四、SQL 脚本执行顺序

```bash
# 1. 执行建表脚本
mysql -u root -p your_database < sql/01_ddl/001_blade_xxx.sql

# 2. 执行字典配置
mysql -u root -p your_database < sql/02_dict/dict_xxx.sql

# 3. 执行菜单权限配置
mysql -u root -p your_database < sql/03_menu/menu_xxx.sql

# 4. 刷新 Redis 缓存（重要！）
redis-cli FLUSHDB  # 或重启 Redis
```

## 五、检查清单

### 字典配置检查
- [ ] 确认是否需要字典配置
- [ ] 系统字典 or 业务字典已确认
- [ ] 字典编码（code）已确认
- [ ] 父级 dict_key = '-1'
- [ ] 子级 dict_key 为实际业务值
- [ ] 字典 SQL 脚本已生成
- [ ] DictEnum 枚举已添加

### 菜单配置检查
- [ ] 父级菜单 ID 已确认
- [ ] 菜单 code 与权限码一致
- [ ] 一级菜单 category = 1
- [ ] 按钮菜单 category = 2
- [ ] action 值正确（1=工具栏，2=操作栏，3=工具操作栏）
- [ ] 权限码命名已确认
- [ ] 菜单 SQL 脚本已生成
