## 阶段 0: 开发前分析

### 0.1 确定模块基本信息

在开始开发前，明确以下信息：

| 项目 | 值 | 说明 |
|------|-----|------|
| 模块名称 | 内部消息 | 中文名称，用于菜单显示 |
| 模块代码 | `innermessage` | 英文标识，用于权限码、路由等 |
| 所属模块目录 | `desk`（工作台） | 决定后端包路径和前端路由前缀 |
| 数据库表名 | `blade_inner_message` | 遵循 BladeX 命名规范 |
| 实体类名 | `InnerMessage` | Java 类名（大驼峰） |
| 前端路由 | `/desk/innermessage` | 浏览器 URL 路径 |

### 0.2 确认技术栈

**后端**:
- Spring Boot 3.2.10 + BladeX 4.8.0
- MyBatis-Plus（数据访问层）
- BladeX 封装的 CRUD 基类

**前端**:
- Vue 3 + Vite
- Element Plus + Avue（低代码表单/表格组件）
- Axios（HTTP 请求）

## 阶段 1: 数据库设计

### 1.1 表设计规范

遵循 BladeX 规范，**主表必须包含以下字段**：

| 字段名 | 类型 | 说明 | 默认值 | 是否必填 |
|--------|------|------|--------|---------|
| `id` | bigint | 主键（雪花算法） | - | 是 |
| `tenant_id` | varchar(12) | 租户 ID | '000000' | 是 |
| `create_user` | bigint | 创建人 | NULL | 否 |
| `create_dept` | bigint | 创建部门 | NULL | 否 |
| `create_time` | datetime | 创建时间 | NULL | 否 |
| `update_user` | bigint | 修改人 | NULL | 否 |
| `update_time` | datetime | 修改时间 | NULL | 否 |
| `status` | int | 状态 | 1 | 否 |
| `is_deleted` | int | 是否删除 | 0 | 否 |

### 1.2 业务字段设计

根据业务需求添加字段：

| 字段名 | 类型 | 说明 | 是否需要字典 |
|--------|------|------|-------------|
| `title` | varchar(255) | 标题 | 否 |
| `message_type` | int | 消息类型 | **是** → 系统字典 |
| `release_time` | datetime | 发布时间 | 否 |
| `message_content` | varchar(2000) | 消息内容 | 否 |


## 阶段 2: 字典与权限配置分析 ⭐

> **为什么这一步很重要？**
> - 字典配置决定前端下拉框的数据来源
> - 权限配置决定 Controller 的 `@PreAuth` 注解和菜单编号
> - 提前生成 SQL 脚本，统一托管执行，避免遗漏

### 2.1 字典配置决策树

```
是否需要下拉选择？
    │
    ├── NO → 跳过字典配置
    │
    └── YES → 继续分析
            │
            ├── 系统字典（推荐）
            │   - 可复用的字典（如：消息类型、订单状态）
            │   - 存储在 blade_dict 表
            │   - 前端通过 /blade-system/dict/dictionary 获取
            │
            └── 业务字典
                - 仅当前模块使用
                - 可存储在前端配置或独立字典表
```

### 2.2 系统字典 SQL 脚本

**文件**: `sql/02_dict/dict_inner_message.sql`

> **注意**: BladeX 字典表结构说明
> - `code`: 字典编码（如 'message_type'）
> - `dict_key`: 字典键值（**顶级节点使用 -1**）
> - `dict_value`: 字典显示名称
> - `parent_id`: 父主键（顶级节点为 0，子节点指向父级 id）
> - 无 `tenant_id`、`create_time` 字段

```sql
-- =============================================
-- 字典：消息类型
-- 字典编码：message_type
-- 所属模块：内部消息
-- =============================================

-- 1. 插入字典类型（父级，dict_key = -1）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES (100001, 0, 'message_type', '-1', '消息类型', 1, NULL, 0, 1, 0);

-- 2. 插入字典值（子级，dict_key 为实际业务值）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES 
(100002, 100001, 'message_type', '1', '通知', 1, NULL, 0, 1, 0),
(100003, 100001, 'message_type', '2', '公告', 2, NULL, 0, 1, 0),
(100004, 100001, 'message_type', '3', '待办', 3, NULL, 0, 1, 0);
```

**字段说明**:
| 字段名 | 说明 | 示例 |
|--------|------|------|
| `id` | 主键（雪花算法） | 100001 |
| `parent_id` | 父主键（顶级为 0，子级指向父级 id） | 0 / 100001 |
| `code` | 字典编码（模块唯一标识） | 'message_type' |
| `dict_key` | 字典键值（**顶级固定为 -1**，子级为业务值） | -1 / 1 / 2 / 3 |
| `dict_value` | 字典显示名称 | '消息类型' / '通知' |
| `sort` | 排序 | 1, 2, 3... |
| `is_sealed` | 是否封存（0: 否，1: 是） | 0 |
| `status` | 状态（默认 1） | 1 |
| `is_deleted` | 是否删除（0: 否，1: 是） | 0 |

**参考现有字典数据**:
- 性别字典：`sex`，父级 `dict_key = '-1'`，子级 `dict_key = '1'` (男), `'2'` (女)
- 通知类型：`notice`，父级 `dict_key = '-1'`，子级 `dict_key = '1'`~`'6'`

### 2.4 权限配置 SQL 脚本

**文件**: `sql/03_menu/menu_inner_message.sql`

> **注意**: BladeX 菜单表结构说明
> - 无自增 ID，使用**雪花算法生成** bigint 类型 ID
> - `parent_id`: 父级菜单 ID（顶级菜单为 0，子菜单指向父级 ID）
> - `category`: 菜单类型（1=一级菜单，2=二级/按钮菜单）
> - `action`: 操作按钮类型（1=工具栏，2=操作栏，3=工具操作栏）
> - 无 `create_time`、`menu_name`、`menu_code`、`menu_uri`、`menu_type`、`menu_icon`、`component_path`、`is_inner`、`is_datascope`、`permission` 字段

**菜单层级关系**:
```
顶级菜单 (parent_id = 0)
  └── 一级菜单 (parent_id = 顶级菜单 ID)
      └── 按钮/功能菜单 (parent_id = 一级菜单 ID)
```

**所属模块查找**:
- 如果模块有父级（如 `desk`），需要查找对应父级 ID
- 参考现有数据：`desk` 工作台 ID = `1123598815738675201`

### 3.8 数据包装类（Wrapper）

**文件**: `src/main/java/org/springblade/modules/desk/wrapper/InnerMessageWrapper.java`

```java
package org.springblade.modules.desk.wrapper;

import java.util.Objects;

import org.springblade.common.cache.DictCache;
import org.springblade.common.enums.DictEnum;
import org.springblade.core.mp.support.BaseEntityWrapper;
import org.springblade.core.tool.utils.BeanUtil;
import org.springblade.modules.desk.pojo.entity.InnerMessage;
import org.springblade.modules.desk.pojo.vo.InnerMessageVO;

public class InnerMessageWrapper extends BaseEntityWrapper<InnerMessage, InnerMessageVO> {

    public static InnerMessageWrapper build() {
        return new InnerMessageWrapper();
    }

    @Override
    public InnerMessageVO entityVO(InnerMessage innerMessage) {
        InnerMessageVO innerMessageVO = Objects.requireNonNull(BeanUtil.copyProperties(innerMessage, InnerMessageVO.class));
        String messageTypeName = DictCache.getValue(DictEnum.MESSAGE_TYPE, innerMessageVO.getMessageType());
        innerMessageVO.setMessageTypeName(messageTypeName);
        return innerMessageVO;
    }

}
```

**要点**:
- ✅ 继承 `BaseEntityWrapper<Entity, VO>`
- ✅ `entityVO()` 方法用于 Entity → VO 转换
- ✅ 使用 `DictCache` 转换字典值（如 `messageType` → `messageTypeName`）
- ✅ 使用阶段 2.2 中定义的 `DictEnum.MESSAGE_TYPE`

```java
import org.springblade.core.mp.support.Condition;
import org.springblade.core.mp.support.Query;
import org.springblade.core.secure.annotation.PreAuth;

import org.springblade.core.tool.api.R;

import org.springblade.modules.desk.pojo.entity.InnerMessage;
import org.springblade.modules.desk.pojo.vo.InnerMessageVO;
import org.springblade.modules.desk.service.IInnerMessageService;
import org.springblade.modules.desk.wrapper.InnerMessageWrapper;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import com.baomidou.mybatisplus.core.metadata.IPage;
import com.github.xiaoymin.knife4j.annotations.ApiOperationSupport;

import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.Parameters;
import io.swagger.v3.oas.annotations.enums.ParameterIn;
import io.swagger.v3.oas.annotations.media.Schema;
import io.swagger.v3.oas.annotations.tags.Tag;
import lombok.AllArgsConstructor;

@TenantDS
@RestController
@AllArgsConstructor
@PreAuth(menu = "innermessage")  // ← 使用阶段 2.4 定义的权限码
@RequestMapping(AppConstant.APPLICATION_DESK_NAME + "/innermessage")
@Tag(name = "内部消息", description = "内部消息接口")
public class InnerMessageController {
    private final IInnerMessageService innerMessageService;
    
    /**
     * 详情
     */
    @GetMapping("/detail")
    @ApiOperationSupport(order = 1)
    @Operation(summary = "详情", description = "传入 innerMessage")
    public R<InnerMessageVO> detail(InnerMessage innerMessage) {
        InnerMessage detail = innerMessageService.getOne(Condition.getQueryWrapper(innerMessage));
        return R.data(InnerMessageWrapper.build().entityVO(detail));
    }
    
    /**
     * 分页
     */
    @GetMapping("/list")
    @Parameters({
        @Parameter(name = "category", description = "公告类型", in = ParameterIn.QUERY, schema = @Schema(type = "integer")),
        @Parameter(name = "title", description = "公告标题", in = ParameterIn.QUERY, schema = @Schema(type = "string"))
    })
    @ApiOperationSupport(order = 2)
    @Operation(summary = "分页", description = "传入 notice")
    public R<IPage<InnerMessageVO>> list(@Parameter(hidden = true) @RequestParam Map<String, Object> innerMessage, Query query) {
        IPage<InnerMessage> pages = innerMessageService.page(Condition.getPage(query), Condition.getQueryWrapper(innerMessage, InnerMessage.class));
        return R.data(InnerMessageWrapper.build().pageVO(pages));
    }
    
    /**
     * 新增
     */
    @PostMapping("/save")
    @ApiOperationSupport(order = 4)
    @Operation(summary = "新增", description = "传入 innerMessage")
    public R save(@RequestBody InnerMessage innerMessage) {
        return R.status(innerMessageService.save(innerMessage));
    }

    /**
     * 修改
     */
    @PostMapping("/update")
    @ApiOperationSupport(order = 5)
    @Operation(summary = "修改", description = "传入 innerMessage")
    public R update(@RequestBody InnerMessage innerMessage) {
        return R.status(innerMessageService.updateById(innerMessage));
    }

    /**
     * 新增或修改
     */
    @XssIgnore
    @PostMapping("/submit")
    @ApiOperationSupport(order = 6)
    @Operation(summary = "新增或修改", description = "传入 innerMessage")
    public R submit(@RequestBody InnerMessage innerMessage) {
        return R.status(innerMessageService.saveOrUpdate(innerMessage));
    }

    /**
     * 删除
     */
    @PostMapping("/remove")
    @ApiOperationSupport(order = 7)
    @Operation(summary = "逻辑删除", description = "传入 innerMessage")
    public R remove(@Parameter(description = "主键集合") @RequestParam String ids) {
        boolean temp = innerMessageService.deleteLogic(Func.toLongList(ids));
        return R.status(temp);
    }

}
```

**要点**:
- ✅ `@TenantDS`：启用多租户数据源
- ✅ `@PreAuth(menu = "innermessage")`：权限控制（**必须与阶段 2.4 定义的菜单码一致**）
- ✅ `@RequestMapping(AppConstant.APPLICATION_DESK_NAME + "/innermessage")`：统一前缀
- ✅ `@Tag` 和 `@Operation`：Swagger 文档注解
- ✅ `Condition.getQueryWrapper()`：自动构建查询条件
- ✅ `Condition.getPage()`：自动处理分页参数
