## 一、核心经验：优先使用 MyBatis-Plus 而非原生 MyBatis

### 问题描述

在初始开发中，使用了原生 MyBatis XML 自定义 SQL 来实现分页查询和字典值转换：

```xml
<!-- 初始实现：AffairMapper.xml -->
<select id="selectAffairPage" resultMap="affairVOResultMap">
    SELECT a.*,
           dict_affair_type.dict_value AS affairTypeDict,
           dict_affair_status.dict_value AS statusDict
    FROM blade_affair a
    LEFT JOIN blade_dict dict_affair_type ON ...
    LEFT JOIN blade_dict dict_affair_status ON ...
    WHERE a.is_deleted = 0
    ...
</select>
```

### 正确做法

参考 innermessage 模块的实现，应该：

1. **使用 MyBatis-Plus 的 BaseMapper** - 继承即可获得 CRUD 能力
2. **使用 Wrapper 类进行字典转换** - 在 Service 层使用 `DictCache.getValue()` 转换
3. **避免 XML 自定义 SQL** - 除非复杂多表关联查询

### 重构后的实现

**1. 创建 AffairWrapper 类**

```java
public class AffairWrapper extends BaseEntityWrapper<Affair, AffairVO> {
    
    public static AffairWrapper build() {
        return new AffairWrapper();
    }

    @Override
    public AffairVO entityVO(Affair affair) {
        AffairVO affairVO = BeanUtil.copyProperties(affair, AffairVO.class);
        // 使用 DictCache 转换字典值
        affairVO.setAffairTypeDict(
            DictCache.getValue(DictEnum.AFFAIR_TYPE, affairVO.getAffairType())
        );
        affairVO.setStatusDict(
            DictCache.getValue(DictEnum.AFFAIR_STATUS, affairVO.getStatus())
        );
        return affairVO;
    }
    
    // 详情 VO 转换（含材料列表）
    public AffairDetailVO entityDetailVO(Affair affair, List<AffairMaterial> materials) {
        // ... 转换逻辑
    }
}
```

**2. 修改 DictEnum 枚举**

```java
public enum DictEnum {
    // ... 其他字典
    AFFAIR_TYPE("affair_type"),      // 新增
    AFFAIR_STATUS("affair_status"),  // 新增
    MATERIAL_TYPE("material_type");  // 新增
}
```

**3. Controller 使用 Wrapper**

```java
@GetMapping("/list")
public R<IPage<AffairVO>> list(Map<String, Object> params, Query query) {
    // 使用 MyBatis-Plus 标准分页
    IPage<Affair> affairPage = affairService.page(
        Condition.getPage(query), 
        Condition.getQueryWrapper(params, Affair.class)
    );
    // 使用 Wrapper 转换字典值
    return R.data(AffairWrapper.build().pageVO(affairPage));
}
```

**4. Service 使用 Wrapper**

```java
@Service
public class AffairServiceImpl extends BaseServiceImpl<AffairMapper, Affair> 
    implements IAffairService {

    @Override
    public AffairDetailVO getDetail(Long id) {
        Affair affair = getById(id);
        List<AffairMaterial> materials = Db.lambdaQuery(AffairMaterial.class)
            .eq(AffairMaterial::getAffairId, id)
            .orderByAsc(AffairMaterial::getSort)
            .list();
        // 使用 Wrapper 转换
        return AffairWrapper.build().entityDetailVO(affair, materials);
    }
}
```

**5. Mapper 只继承 BaseMapper**

```java
public interface AffairMapper extends BaseMapper<Affair> {
    // 无自定义方法
}
```

## 二、BladeX 标准开发模式

### 完整流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Entity (实体类)                                          │
│    - 继承 BaseEntity（包含审计字段）                         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. VO (视图对象)                                             │
│    - 继承 Entity 或独立定义                                  │
│    - 包含字典字段（如 affairTypeDict）                       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. Mapper                                                    │
│    - 继承 BaseMapper<Entity>                                 │
│    - 无需 XML（除非复杂查询）                                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. Service                                                   │
│    - 继承 BaseServiceImpl<Mapper, Entity>                    │
│    - 使用 Db.lambdaQuery() 进行复杂查询                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. Wrapper (包装类)                                          │
│    - 继承 BaseEntityWrapper<Entity, VO>                      │
│    - 实现 entityVO() 方法进行字典转换                        │
│    - 使用 DictCache.getValue() 获取字典值                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. Controller                                                │
│    - 继承 BladeController                                    │
│    - 使用 Wrapper.build().pageVO() 转换分页结果             │
│    - 使用 Wrapper.build().entityVO() 转换单条数据           │
└─────────────────────────────────────────────────────────────┘
```

## 三、DictCache 使用规范

### 添加字典枚举

在 `DictEnum.java` 中添加：

```java
/**
 * 政务服务事项类别
 */
AFFAIR_TYPE("affair_type"),
/**
 * 政务服务事项状态
 */
AFFAIR_STATUS("affair_status"),
/**
 * 所需材料类型
 */
MATERIAL_TYPE("material_type");
```

### 使用 DictCache 转换

```java
//  Integer 类型字典键
String typeName = DictCache.getValue(DictEnum.AFFAIR_TYPE, affair.getAffairType());

//  String 类型字典键
String statusName = DictCache.getValue(DictEnum.AFFAIR_STATUS, String.valueOf(affair.getStatus()));
```

## 四、重构前后对比

| 项目 | 重构前 | 重构后 |
|------|--------|--------|
| 查询方式 | XML 自定义 SQL | MyBatis-Plus QueryWrapper |
| 字典转换 | SQL JOIN 关联 | DictCache 缓存查询 |
| Mapper | 有 selectAffairPage 方法 | 仅继承 BaseMapper |
| XML | 有 AffairMapper.xml | 已删除 |
| Service | 手动构建 VO | Wrapper 自动转换 |
| 性能 | 每次查询 JOIN 字典表 | 字典缓存，性能更优 |
