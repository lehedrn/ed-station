## 二、前端开发问题

### 问题 1：前端页面文件结构导致路由加载失败 ⭐⭐⭐

**问题描述**：
- 访问 `http://localhost:3888/affair/manage` 时页面显示空白
- 浏览器控制台无报错，但页面内容未渲染

**原因分析**：
- BladeX 框架的 `avue-router.js` 中 `formatPath` 函数通过以下逻辑拼接组件路径：
  ```javascript
  ele.component = 'views' + ele[propsDefault.path];
  ```
- 如果后端菜单配置的 `path` 为 `/affair/manage`，则期望的组件路径为：
  ```
  views/affair/manage.vue
  ```
- 但实际文件结构是：
  ```
  views/affair/manage/index.vue  （嵌套结构）
  ```
- `import.meta.glob` 无法正确匹配 `index.vue` 结构

**对比参考**：
```
✅ innermessage 模块（正常）：
src/views/desk/innermessage.vue

❌ affair 模块（问题）：
src/views/affair/manage/index.vue
```

**解决方案**：
1. 将 `views/affair/manage/index.vue` 重命名为 `views/affair/manage.vue`
2. 将子组件 `views/affair/manage/components/MaterialDialog.vue` 移到同级目录
3. 修改组件导入路径：
   ```javascript
   // 修改前
   import MaterialDialog from './components/MaterialDialog.vue';
   
   // 修改后
   import MaterialDialog from './MaterialDialog.vue';
   ```

**修复后的文件结构**：
```
src/views/affair/
├── manage.vue           (主页面)
└── MaterialDialog.vue   (子组件)
```

**经验教训**：
> ⚠️ **BladeX 框架的前端页面必须采用扁平结构**，不能使用 `index.vue` 嵌套结构。
> 参考已正常工作的模块（如 `innermessage`），保持文件结构一致。

## 三、联调测试问题

### 问题 3：cURL 测试中中文参数导致 400 错误 ⭐⭐

**问题描述**：
- 测试脚本执行 `T01-02: 按事项名称筛选` 时返回 400 错误
- URL: `GET /blade-affair/affair/list?current=1&size=10&affairName=测试`

**原因分析**：
- URL 中的中文字符未进行 URL 编码
- 某些环境下（尤其是 WSL2）curl 对非 ASCII 字符处理不一致

**解决方案**：
```bash
# 使用 Python 进行 URL 编码
AFFAIR_NAME_ENCODED=$(python3 -c "import urllib.parse; print(urllib.parse.quote('测试', encoding='utf-8'))")
RESPONSE=$(curl -s -X GET "$BASE_URL/blade-affair/affair/list?current=1&size=10&affairName=${AFFAIR_NAME_ENCODED}" ...)
```

**经验教训**：
> ⚠️ **cURL 测试脚本中的中文参数必须进行 URL 编码**，建议使用 Python 的 `urllib.parse.quote()` 函数。

### 问题 4：新增接口返回空对象无法获取 ID ⭐⭐

**问题描述**：
- 新增事项接口返回 `{ "code": 200, "success": true, "data": {} }`
- 测试脚本无法获取生成的 ID 进行后续测试

**原因分析**：
- BladeX 框架的 `save` 接口返回类型是 `R<Boolean>`，不是 `R<Long>`
- 这是框架设计，不是 bug

**解决方案**：
- 通过时间戳精确匹配查询列表获取 ID：
  ```bash
  TIMESTAMP=$(date +%s)
  # 创建事项时包含时间戳
  CREATE_DATA='{"affairName":"测试事项_'$TIMESTAMP'", ...}'
  
  # 通过列表查询获取 ID
  LIST_RESPONSE=$(curl -s -X GET "$BASE_URL/blade-affair/affair/list?current=1&size=1&affairName=测试事项_$TIMESTAMP" ...)
  TEST_AFFAIR_ID=$(echo "$LIST_RESPONSE" | jq -r '.data.records[0].id // empty')
  ```

**经验教训**：
> ⚠️ **BladeX 的 save 接口不返回生成的 ID**，需要通过列表查询间接获取。
> 建议在测试数据命名中包含时间戳，以便精确匹配。

### 问题 5：数据库字符集排序规则冲突 ⭐⭐⭐

**问题描述**：
- 执行分页查询时报错：
  ```
  Illegal mix of collations (utf8mb4_0900_ai_ci,IMPLICIT) and (utf8mb4_general_ci,IMPLICIT)
  ```

**原因分析**：
- `blade_affair` 表的字符集是 `utf8mb4_0900_ai_ci`
- `blade_dict` 表的字符集是 `utf8mb4_general_ci`
- 两表 JOIN 时字符集不兼容

**解决方案**：
在 `AffairMapper.xml` 的 JOIN 语句中显式指定 COLLATE：
```xml
<!-- 修改前 -->
LEFT JOIN blade_dict dict_affair_type ON a.affair_type = dict_affair_type.dict_key

<!-- 修改后 -->
LEFT JOIN blade_dict dict_affair_type ON a.affair_type COLLATE utf8mb4_general_ci = dict_affair_type.dict_key COLLATE utf8mb4_general_ci
```

**经验教训**：
> ⚠️ **涉及多表 JOIN 时，如果字符集不一致，必须在 ON 条件中使用 COLLATE 显式转换**。
> 或者统一所有表的字符集为 `utf8mb4_0900_ai_ci`。

## 四、测试脚本问题

### 问题 6：Playwright 测试选择器匹配多个元素 ⭐

**问题描述**：
- 测试报错：`strict mode violation: locator('.avue-crud, .basic-container') resolved to 2 elements`
- 页面中有多个匹配相同选择器的元素

**原因分析**：
- 页面嵌套结构导致多个 `.basic-container` 元素存在
- Playwright 的严格模式不允许选择器匹配多个元素

**解决方案（待实施）**：
```javascript
// 使用更精确的选择器
const crudContainer = page.locator('.avue-crud.avue-crud--card');
// 或使用 getByText/GetByRole 等定位方式
const crudContainer = page.getByText('事项名称:').first();
```

## 五、最佳实践总结

### 前端开发

1. **文件结构规范**：
   ```
   ✅ 推荐：src/views/module/page.vue
   ❌ 避免：src/views/module/page/index.vue
   ```

2. **子组件位置**：
   ```
   ✅ 推荐：src/views/module/Component.vue (与主页面同级)
   ```

3. **组件导入路径**：
   ```javascript
   // 同级目录
   import ChildComponent from './ChildComponent.vue';
   
   // 父目录
   import ParentComponent from '../ParentComponent.vue';
   ```

### 后端开发

1. **字符集统一**：
   - 所有表使用 `utf8mb4_0900_ai_ci`
   - JOIN 时使用 COLLATE 显式转换

2. **接口返回规范**：
   - 遵循 BladeX 的 `R<T>` 泛型返回
   - save 接口返回 `R<Boolean>` 而非 ID

### 测试脚本

1. **cURL 测试**：
   - 中文参数使用 `urllib.parse.quote()` 编码
   - 通过时间戳精确匹配获取生成数据

2. **Playwright 测试**：
   - 使用精确选择器避免 strict mode violation
   - 测试前准备充足的测试数据
