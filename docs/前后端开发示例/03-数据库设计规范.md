# 数据库设计规范

> BladeX 框架下的数据库表结构设计规范

---

## 一、表设计规范

### 1.1 主表必须字段

遵循 BladeX 规范，**主表必须包含以下字段**：

| 字段名 | 类型 | 说明 | 默认值 | 是否必填 |
|--------|------|------|--------|---------|
| `id` | bigint | 主键（雪花算法） | - | 是 |
| `tenant_id` | varchar(12) | 租户 ID | '000000' | 是 |
| `create_user` | bigint | 创建人 | NULL | 否 |
| `create_dept` | bigint | 创建部门 | NULL | 否 |
| `create_time` | datetime | 创建时间 | NULL | 否 |
| `update_user` | bigint | 修改人 | NULL | 否 |
| `update_time` | datetime | 修改时间 | NULL | 否 |
| `status` | int | 状态 | 1 | 否 |
| `is_deleted` | int | 是否删除 | 0 | 否 |

### 1.2 业务字段设计

根据业务需求添加字段，注意：

| 字段类型 | 命名规范 | 示例 |
|---------|---------|------|
| 布尔类型 | is_xxx | is_urgent, is_read |
| 日期时间 | xxx_time | release_time, expire_time |
| 数量类型 | xxx_count | view_count, like_count |
| 类型字段 | xxx_type | message_type, order_type |
| 金额字段 | xxx_amount / xxx_fee | total_amount, service_fee |
| 名称字段 | xxx_name | user_name, dept_name |
| 编号字段 | xxx_code / xxx_no | order_code, user_no |

### 1.3 索引规范

**必须添加的索引**：
- `idx_tenant`: 租户 ID 索引（多租户查询优化）
- `idx_status`: 状态索引（状态查询优化）
- `idx_deleted`: 删除标记索引（逻辑删除查询优化）

**建议添加的索引**：
- 常用查询字段
- 关联查询字段
- 排序字段

## 二、SQL 建表脚本示例

### 2.1 完整建表脚本

```sql
-- =============================================
-- 模块：内部消息
-- 表名：blade_inner_message
-- 创建时间：2026-04-02
-- =============================================

DROP TABLE IF EXISTS `blade_inner_message`;
CREATE TABLE `blade_inner_message`  (
  `id` bigint NOT NULL COMMENT '主键',
  `tenant_id` varchar(12) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci NULL DEFAULT '000000' COMMENT '租户 ID',
  `title` varchar(255) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci NULL DEFAULT NULL COMMENT '标题',
  `message_type` int NULL DEFAULT NULL COMMENT '消息类型',
  `release_time` datetime NULL DEFAULT NULL COMMENT '发布时间',
  `message_content` varchar(2000) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci NULL DEFAULT NULL COMMENT '内容',
  `create_user` bigint NULL DEFAULT NULL COMMENT '创建人',
  `create_dept` bigint NULL DEFAULT NULL COMMENT '创建部门',
  `create_time` datetime NULL DEFAULT NULL COMMENT '创建时间',
  `update_user` bigint NULL DEFAULT NULL COMMENT '修改人',
  `update_time` datetime NULL DEFAULT NULL COMMENT '修改时间',
  `status` int NULL DEFAULT 1 COMMENT '状态',
  `is_deleted` int NULL DEFAULT 0 COMMENT '是否已删除',
  PRIMARY KEY (`id`) USING BTREE,
  INDEX `idx_tenant` (`tenant_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_deleted` (`is_deleted`)
) ENGINE = InnoDB CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci COMMENT = '内部消息';
```

### 2.2 脚本文件命名

- 位置：`ed-station-boot/sql/01_ddl/`
- 命名：`001_blade_xxx.sql`（序号_表名.sql）
- 多个表按执行顺序编号

## 三、字段类型选择

### 3.1 字符串类型

| 类型 | 长度 | 使用场景 |
|------|------|---------|
| varchar(12) | 12 | 租户 ID、手机号 |
| varchar(32) | 32 | 短编码、哈希值 |
| varchar(50) | 50 | 名称、标题 |
| varchar(100) | 100 | 较长描述 |
| varchar(255) | 255 | 通用字符串 |
| varchar(500) | 500 | 较长内容 |
| varchar(2000) | 2000 | 大段文本（接近 limit） |
| text | 65535 | 超大文本（少用） |

### 3.2 数值类型

| 类型 | 范围 | 使用场景 |
|------|------|---------|
| int | 0~21 亿 | 数量、类型、状态 |
| bigint | 超大数 | 金额（分）、累计值 |
| decimal(10,2) | 金额 | 货币金额（元） |

### 3.3 日期类型

| 类型 | 格式 | 使用场景 |
|------|------|---------|
| datetime | YYYY-MM-DD HH:mm:ss | 时间戳 |
| date | YYYY-MM-DD | 纯日期 |
| time | HH:mm:ss | 纯时间 |

## 四、注释规范

### 4.1 表注释

```sql
COMMENT = '模块名_功能描述'
-- 示例：COMMENT = '内部消息表'
```

### 4.2 字段注释

```sql
`field_name` type NULL DEFAULT NULL COMMENT '字段说明'
-- 示例：`message_type` int NULL DEFAULT NULL COMMENT '消息类型'
```

### 4.3 字典字段注释

```sql
`status` int NULL DEFAULT NULL COMMENT '状态（字典：common_status）'
```

## 五、检查清单

### 建表前检查
- [ ] 表名遵循 `blade_模块_功能` 命名
- [ ] 主键类型为 bigint（雪花算法）
- [ ] 包含所有 BladeX 规范字段
- [ ] 租户 ID 默认值为 '000000'
- [ ] 字符集统一为 utf8mb4

- [ ] 主键索引已创建
- [ ] 租户索引已创建
- [ ] 常用查询字段有索引
- [ ] 表注释已添加
- [ ] 字段注释已添加
