# 接口公约规范

> BladeX 框架下前后端接口的统一传参规范和响应格式

---

## 一、后端接口响应结构

### 1.1 统一响应格式 `R<T>`

所有 Controller 接口返回统一的 `R<T>` 对象，结构如下：

```java
{
  "code": 200,           // 状态码
  "msg": "操作成功",      // 响应消息
  "data": { ... }        // 业务数据（泛型）
}
```

### 1.2 R 类常用方法

| 方法 | 说明 | 返回结构 | 使用场景 |
|------|------|----------|---------|
| `R.data(T data)` | 返回业务数据 | `{"code": 200, "data": {...}}` | 查询接口（详情、列表） |
| `R.status(boolean success)` | 返回操作状态 | `{"code": 200, "msg": "操作成功"}` | 新增、修改、删除 |
| `R.success()` | 返回成功 | `{"code": 200, "msg": "成功"}` | 无业务数据的成功响应 |
| `R.fail(String msg)` | 返回失败 | `{"code": 非 200, "msg": "失败原因"}` | 业务失败响应 |

### 1.3 后端接口示例

```java
// 详情接口 - 返回单个对象
@GetMapping("/detail")
public R<InnerMessageVO> detail(InnerMessage innerMessage) {
    InnerMessage detail = innerMessageService.getOne(...);
    return R.data(InnerMessageWrapper.build().entityVO(detail));
}
// 响应：{"code": 200, "data": {"id": 123, "title": "标题", ...}}

// 分页接口 - 返回分页对象
@GetMapping("/list")
public R<IPage<InnerMessageVO>> list(Map<String, Object> params, Query query) {
    IPage<InnerMessage> pages = innerMessageService.page(...);
    return R.data(InnerMessageWrapper.build().pageVO(pages));
}
// 响应：{"code": 200, "data": {"records": [...], "total": 100, "current": 1, "size": 10}}

// 新增/修改/删除 - 返回状态
@PostMapping("/submit")
public R submit(@RequestBody InnerMessage innerMessage) {
    return R.status(innerMessageService.saveOrUpdate(innerMessage));
}
// 响应：{"code": 200, "msg": "操作成功"}
```

## 二、后端接口传参规范

### 2.1 接口类型与传参方式

| 接口类型 | HTTP 方法 | 传参方式 | 注解 | 示例 |
|---------|----------|---------|------|------|
| 详情查询 | GET | 查询参数 | `@RequestParam` (隐式) | `/detail?id=123` |
| 分页列表 | GET | 查询参数 | `@RequestParam Map + Query` | `/list?current=1&size=10&title=xxx` |
| 新增 | POST | JSON Body | `@RequestBody` | `{"title": "标题", ...}` |
| 修改 | POST | JSON Body | `@RequestBody` | `{"id": 123, "title": "新标题", ...}` |
| 删除 | POST | 查询参数 | `@RequestParam` | `/remove?ids=123,124,125` |
| 提交 (新增或修改) | POST | JSON Body | `@RequestBody` | `{"id": 123, ...}` |

### 2.2 分页参数规范

使用 BladeX 封装的 `Query` 对象接收分页参数：

```java
@GetMapping("/list")
public R<IPage<VO>> list(@RequestParam Map<String, Object> params, Query query) {
    // Query 对象自动接收：
    // - current: 当前页码（从 1 开始）
    // - size: 每页条数
    // - sortField: 排序字段
    // - sortOrder: 排序方式（asc/desc）
}
```

**前端传参示例**：
```javascript
{
  current: 1,      // 当前页
  size: 10,        // 每页条数
  title: '测试',   // 业务查询参数
  messageType: 1   // 业务查询参数
}
```

### 2.3 路径变量规范

```java
// 路径变量（较少使用）
@GetMapping("/info/{id}")
public R<VO> info(@PathVariable Long id) { ... }
```

### 2.4 批量操作参数

```java
// 批量删除 - ids 为逗号分隔的字符串
@PostMapping("/remove")
public R remove(@RequestParam String ids) {
    List<Long> idList = Func.toLongList(ids); // 123,124,125 → [123, 124, 125]
    ...
}
```

## 三、前端 API 封装规范

### 3.1 标准 API 模块结构

```javascript
import request from '@/axios';

// 分页查询
export const getList = (current, size, params) => {
  return request({
    url: '/blade-desk/innermessage/list',
    method: 'get',
    params: { ...params, current, size },
  });
};

// 详情
export const getDetail = id => {
  return request({
    url: '/blade-desk/innermessage/detail',
    method: 'get',
    params: { id },
  });
};

// 删除
export const remove = ids => {
  return request({
    url: '/blade-desk/innermessage/remove',
    method: 'post',
    params: { ids },
  });
};

// 新增
export const add = row => {
  return request({
    url: '/blade-desk/innermessage/submit',
    method: 'post',
    data: row,
  });
};

// 修改
export const update = row => {
  return request({
    url: '/blade-desk/innermessage/submit',
    method: 'post',
    data: row,
  });
};
```

### 3.2 请求配置选项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `url` | string | - | 请求地址（自动添加 `/api` 前缀） |
| `method` | string | 'get' | 请求方法 |
| `params` | object | - | URL 查询参数（GET 请求） |
| `data` | object | - | 请求体（POST 请求） |
| `cryptoToken` | boolean | false | 是否加密 Token |
| `cryptoData` | boolean | false | 是否加密请求数据 |
| `meta.isToken` | boolean | false | 是否需要 Token |
| `meta.isSerialize` | boolean | false | 是否表单序列化 |

### 3.3 响应数据处理

前端 axios 拦截器自动处理响应：

```javascript
// axios 拦截器自动解析
res => {
  const status = res.data.code;  // 获取状态码
  if (status !== 200) {
    // 自动弹窗提示错误
    ElMessage.error(res.data.msg);
    return Promise.reject(...);
  }
  return res;  // res.data.data 是业务数据
}
```

**前端调用示例**：
```javascript
// 查询详情
getDetail(id).then(res => {
  const data = res.data.data;  // 直接获取业务数据
  this.form = data;
});

// 分页查询
getList(page, size, params).then(res => {
  const pageData = res.data.data;
  this.data = pageData.records;  // 数据列表
  this.page.total = pageData.total;  // 总数
});

// 提交操作
add(row).then(() => {
  this.$message.success('操作成功');
  this.onLoad(this.page);  // 刷新列表
});
```

## 四、前后端接口对照表

### 4.1 标准 CRUD 接口

| 功能 | 前端 API | 后端 URL | 方法 | 参数 | 响应 |
|------|---------|---------|------|------|------|
| 分页查询 | `getList(current, size, params)` | `/list` | GET | Query + Map | `IPage<VO>` |
| 详情查询 | `getDetail(id)` | `/detail` | GET | 实体对象 | `VO` |
| 新增 | `add(row)` | `/submit` | POST | Entity (Body) | `boolean` |
| 修改 | `update(row)` | `/submit` | POST | Entity (Body) | `boolean` |
| 删除 | `remove(ids)` | `/remove` | POST | ids (String) | `boolean` |

### 4.2 接口命名约定

| 后端 URL 后缀 | 前端方法名 | 说明 |
|--------------|-----------|------|
| `/list` | `getList` | 分页列表（推荐） |
| `/page` | `getPage` | 自定义分页（多表联查） |
| `/detail` | `getDetail` / `getXxx` | 详情查询 |
| `/info` | `getInfo` | 当前用户信息 |
| `/submit` | `add` / `update` | 新增或修改（根据有无 id 判断） |
| `/save` | `add` | 新增 |
| `/update` | `update` | 修改 |
| `/remove` | `remove` | 删除（逻辑删除） |
| `/delete` | `remove` | 删除（物理删除，少用） |
| `/enable` / `/disable` | `enable` / `disable` | 启用/禁用 |
| `/grant` | `grant` | 授权 |
| `/reset-password` | `resetPassword` | 重置密码 |

## 五、数据结构规范

### 5.1 Entity 实体类规范

```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("blade_xxx")
@Schema(description = "实体类描述")
public class XxxEntity extends TenantEntity {
    
    @Schema(description = "业务字段")
    private String field;
    
    // 继承 TenantEntity 的字段：
    // - id (bigint, 雪花算法)
    // - tenantId (varchar)
    // - createUser, createDept, createTime (bigint, datetime)
    // - updateUser, updateTime (bigint, datetime)
    // - status (int, 默认 1)
    // - isDeleted (int, 默认 0)
}
```

### 5.2 VO 视图对象规范

```java
@Data
@EqualsAndHashCode(callSuper = true)
@Schema(description = "VO 类描述")
public class XxxVO extends XxxEntity {
    
    // 扩展字段（用于前端展示）
    @Schema(description = "字典名称")
    private String xxxName;
    
    @Schema(description = "创建人名称")
    private String createUserName;
}
```

### 5.3 分页响应结构

```javascript
{
  "code": 200,
  "data": {
    "records": [     // 数据列表
      {"id": 1, "name": "测试1"},
      {"id": 2, "name": "测试2"}
    ],
    "total": 100,    // 总记录数
    "current": 1,    // 当前页
    "size": 10,      // 每页条数
    "pages": 10      // 总页数
  }
}
```

## 六、特殊场景处理

### 6.1 字典值处理

**后端 Wrapper 类处理字典转换**：
```java
@Override
public XxxVO entityVO(XxxEntity entity) {
    XxxVO vo = BeanUtil.copyProperties(entity, XxxVO.class);
    // 字典值转换（如 messageType=1 → messageTypeName="通知"）
    String typeName = DictCache.getValue(DictEnum.MESSAGE_TYPE, vo.getMessageType());
    vo.setMessageTypeName(typeName);
    return vo;
}
```

### 6.2 日期时间范围查询

**前端传参**：
```javascript
{
  releaseTimeRange: ['2026-04-01 00:00:00', '2026-04-02 23:59:59']
}
```

**后端处理**（在 onLoad 或 Service 中转换）：
```javascript
// 前端转换
if (releaseTimeRange) {
  values = {
    ...values,
    releaseTime_datege: releaseTimeRange[0],  // 大于等于
    releaseTime_datelt: releaseTimeRange[1],  // 小于等于
  };
}
```

**Controller 接收**：
```java
// BladeX 的 Condition.getQueryWrapper 自动识别 _datege/_datelt 后缀
// 生成 SQL: release_time >= ? AND release_time <= ?
```

## 七、调试技巧

### 7.1 后端接口调试

1. 访问 Swagger 文档：`http://localhost:18080/doc.html`
2. 查看接口定义和参数
3. 直接在 Swagger 中测试接口

### 7.2 前端接口调试

1. 浏览器开发者工具 → Network 标签
2. 查看请求 URL、方法、参数
3. 查看响应数据结构

### 7.3 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 401 未授权 | Token 过期/无效 | 检查 Token 配置，刷新 Token |
| 403 无权限 | 权限码不匹配 | 检查 `@PreAuth` 和菜单权限配置 |
| 500 服务器错误 | 后端异常 | 查看后端日志 `tail -f logs/backend.log` |
| 数据为空 | 查询条件问题 | 检查查询参数、数据库连接 |
| 字典不显示 | 字典编码错误 | 检查 `dicUrl` 和字典配置 |
