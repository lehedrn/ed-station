# 接口公约规范

> BladeX 框架下前后端接口的统一传参规范和响应格式（基于源码分析）

---

## 一、后端接口响应结构

### 1.1 统一响应格式 `R<T>`

所有 Controller 接口返回统一的 `R<T>` 对象，源码位于 `blade-core-tool/src/main/java/org/springblade/core/tool/api/R.java`。

**响应结构**：
```json
{
  "code": 200,           // 状态码
  "success": true,       // 是否成功（自动计算）
  "data": { ... },       // 业务数据（泛型）
  "msg": "操作成功"      // 响应消息
}
```

**R 类核心属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| `code` | int | 状态码（200=成功，其他=失败） |
| `success` | boolean | 是否成功（自动计算：`code == 200`） |
| `data` | T | 业务数据（泛型） |
| `msg` | String | 响应消息 |

### 1.2 状态码枚举（ResultCode）

源码位于 `blade-core-tool/src/main/java/org/springblade/core/tool/api/ResultCode.java`。

| 状态码 | 名称 | HTTP 状态码 | 说明 |
|--------|------|------------|------|
| `200` | SUCCESS | 200 OK | 操作成功 |
| `400` | FAILURE | Bad Request | 业务异常/请求参数错误 |
| `401` | UN_AUTHORIZED | Unauthorized | 请求未授权 |
| `403` | REQ_REJECT | Forbidden | 请求被拒绝 |
| `404` | NOT_FOUND | Not Found | 404 没找到请求 |
| `500` | INTERNAL_SERVER_ERROR | Internal Server Error | 服务器异常 |

### 1.3 R 类常用方法

| 方法 | 源码实现 | 返回结构 | 使用场景 |
|------|---------|----------|---------|
| `R.data(T data)` | `data(HttpServletResponse.SC_OK, data, msg)` | `{"code": 200, "data": {...}}` | 查询接口（详情、列表） |
| `R.data(T data, String msg)` | `new R<>(code, data, msg)` | `{"code": 200, "data": {...}, "msg": "..."}` | 带消息的查询接口 |
| `R.status(boolean flag)` | `flag ? R.success() : R.fail()` | `{"code": 200, "msg": "操作成功"}` | 新增、修改、删除 |
| `R.success()` | `new R<>(ResultCode.SUCCESS)` | `{"code": 200, "msg": "操作成功"}` | 无数据的成功响应 |
| `R.fail(String msg)` | `new R<>(ResultCode.FAILURE, msg)` | `{"code": 400, "msg": "..."}` | 业务失败响应 |

### 1.4 后端接口示例

```java
// 详情接口 - 返回单个对象
@GetMapping("/detail")
public R<InnerMessageVO> detail(InnerMessage innerMessage) {
    InnerMessage detail = innerMessageService.getOne(Condition.getQueryWrapper(innerMessage));
    return R.data(InnerMessageWrapper.build().entityVO(detail));
}
// 响应：{"code": 200, "success": true, "data": {"id": 123, "title": "标题", ...}, "msg": "操作成功"}

// 分页接口 - 返回分页对象
@GetMapping("/list")
public R<IPage<InnerMessageVO>> list(@RequestParam Map<String, Object> innerMessage, Query query) {
    IPage<InnerMessage> pages = innerMessageService.page(Condition.getPage(query), Condition.getQueryWrapper(innerMessage, InnerMessage.class));
    return R.data(InnerMessageWrapper.build().pageVO(pages));
}
// 响应：{"code": 200, "data": {"records": [...], "total": 100, "current": 1, "size": 10}, "msg": "操作成功"}

// 新增/修改/删除 - 返回状态
@PostMapping("/submit")
public R submit(@RequestBody InnerMessage innerMessage) {
    return R.status(innerMessageService.saveOrUpdate(innerMessage));
}
// 响应：{"code": 200, "success": true, "msg": "操作成功"}
```

## 二、后端接口传参规范

### 2.1 接口类型与传参方式

| 接口类型 | HTTP 方法 | 传参方式 | 注解 | 示例 |
|---------|----------|---------|------|------|
| 详情查询 | GET | 查询参数 | `@RequestParam` (隐式) | `/detail?id=123` |
| 分页列表 | GET | 查询参数 | `@RequestParam Map + Query` | `/list?current=1&size=10&title=xxx` |
| 新增 | POST | JSON Body | `@RequestBody` | `{"title": "标题", ...}` |
| 修改 | POST | JSON Body | `@RequestBody` | `{"id": 123, "title": "新标题", ...}` |
| 删除 | POST | 查询参数 | `@RequestParam` | `/remove?ids=123,124,125` |
| 提交 (新增或修改) | POST | JSON Body | `@RequestBody` | `{"id": 123, ...}` |

### 2.2 分页参数规范（Query 类）

源码位于 `blade-starter-mybatis/src/main/java/org/springblade/core/mp/support/Query.java`。

**Query 类属性**：
| 属性 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `current` | Integer | 当前页码（从 1 开始） | 1 |
| `size` | Integer | 每页条数 | 10 |
| `ascs` | String | 正排序字段（逗号分隔） | - |
| `descs` | String | 倒排序字段（逗号分隔） | - |

**Controller 接收示例**：
```java
@GetMapping("/list")
public R<IPage<VO>> list(@RequestParam Map<String, Object> params, Query query) {
    // Condition.getPage(query) 自动处理分页和排序
    // current 默认 1，size 默认 10
}
```

**前端传参示例**：
```javascript
{
  current: 1,           // 当前页（从 1 开始）
  size: 10,             // 每页条数
  ascs: 'create_time',  // 正排序（可选）
  descs: 'status',      // 倒排序（可选）
  title: '测试',        // 业务查询参数（自动模糊匹配）
  messageType: 1        // 业务查询参数（精确匹配）
}
```

### 2.3 Condition 工具类

源码位于 `blade-starter-mybatis/src/main/java/org/springblade/core/mp/support/Condition.java`。

**核心方法**：
| 方法 | 说明 | 返回值 |
|------|------|--------|
| `getPage(Query query)` | 将 Query 转为 MyBatis-Plus 的 Page 对象 | `IPage<T>` |
| `getQueryWrapper(T entity)` | 将实体类转为 QueryWrapper | `QueryWrapper<T>` |
| `getQueryWrapper(Map query, Class<T> clazz)` | 将 Map 查询条件转为 QueryWrapper | `QueryWrapper<T>` |

**自动过滤规则**：
- 排除 `Authorization`、`current`、`size`、`ascs`、`descs` 等特殊字段
- 只保留与实体类字段名匹配的参数
- 自动过滤 SQL 注入关键词

### 2.4 SqlKeyword 查询条件后缀

源码位于 `blade-starter-mybatis/src/main/java/org/springblade/core/mp/support/SqlKeyword.java`。

**查询条件后缀对照表**：

| 后缀 | 说明 | SQL 示例 | 前端参数名示例 |
|------|------|---------|---------------|
| `_equal` | 等于 | `field = ?` | `messageType_equal=1` |
| `_notequal` | 不等于 | `field != ?` | `status_notequal=0` |
| `_like` | 模糊（前后%） | `field LIKE %?%` | `title_like=测试` |
| `_like_left` | 左模糊 | `field LIKE %?` | `name_like_left=张` |
| `_like_right` | 右模糊 | `field LIKE ?%` | `code_like_right=ABC` |
| `_notlike` | 不包含 | `field NOT LIKE %?%` | `name_notlike=测试` |
| `_ge` | 大于等于 | `field >= ?` | `age_ge=18` |
| `_le` | 小于等于 | `field <= ?` | `score_le=100` |
| `_gt` | 大于 | `field > ?` | `price_gt=100` |
| `_lt` | 小于 | `field < ?` | `discount_lt=1` |
| `_datege` | 日期大于等于 | `field >= ?` | `releaseTime_datege=2026-04-01` |
| `_dategt` | 日期大于 | `field > ?` | `createTime_dategt=2026-01-01` |
| `_dateequal` | 日期等于 | `field = ?` | `birthday_dateequal=1990-01-01` |
| `_datele` | 日期小于等于 | `field <= ?` | `endTime_datele=2026-12-31` |
| `_datelt` | 日期小于 | `field < ?` | `startTime_datelt=2026-01-01` |
| `_null` | 为空 | `field IS NULL` | `remark_null=true` |
| `_notnull` | 不为空 | `field IS NOT NULL` | `email_notnull=true` |
| `_ignore` | 忽略此条件 | - | `temp_ignore=true` |

**日期范围查询示例**：
```javascript
// 前端传参
{
  releaseTime_datege: '2026-04-01 00:00:00',
  releaseTime_datelt: '2026-04-30 23:59:59'
}

// 自动生成 SQL
// WHERE release_time >= '2026-04-01 00:00:00' AND release_time <= '2026-04-30 23:59:59'
```

## 三、前端 API 封装规范

### 3.1 标准 API 模块结构

```javascript
import request from '@/axios';

// 分页查询
export const getList = (current, size, params) => {
  return request({
    url: '/blade-desk/innermessage/list',
    method: 'get',
    params: { ...params, current, size },
    cryptoToken: false,
    cryptoData: false,
  });
};

// 详情
export const getDetail = id => {
  return request({
    url: '/blade-desk/innermessage/detail',
    method: 'get',
    params: { id },
  });
};

// 删除
export const remove = ids => {
  return request({
    url: '/blade-desk/innermessage/remove',
    method: 'post',
    params: { ids },
  });
};

// 新增/修改（共用 submit 接口）
export const add = row => {
  return request({
    url: '/blade-desk/innermessage/submit',
    method: 'post',
    data: row,
  });
};

export const update = row => {
  return request({
    url: '/blade-desk/innermessage/submit',
    method: 'post',
    data: row,
  });
};
```

### 3.2 请求配置选项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `url` | string | - | 请求地址（自动添加 `/api` 前缀） |
| `method` | string | 'get' | 请求方法 |
| `params` | object | - | URL 查询参数（GET 请求） |
| `data` | object | - | 请求体（POST 请求） |
| `cryptoToken` | boolean | false | 是否加密 Token |
| `cryptoData` | boolean | false | 是否加密请求数据 |
| `meta.isToken` | boolean | false | 是否需要 Token |
| `meta.isSerialize` | boolean | false | 是否表单序列化 |
| `text` | boolean | false | 是否纯文本请求（用于加密） |

### 3.3 响应数据处理

前端 axios 拦截器自动处理响应（`axios.js`）：

```javascript
// http response 拦截器
res => {
  const cryptoData = config.cryptoData === true;
  // 解析加密报文
  if (cryptoData) {
    res.data = JSON.parse(crypto.decryptAES(res.data, crypto.aesKey));
  }
  
  // 获取状态码
  const status = res.data.error_code || res.data.code || res.status;
  const message = res.data.msg || res.data.error_description || '系统错误';
  
  // 401 刷新 token 逻辑
  if (status === 401 && !config._retry) {
    // 自动刷新 token 并重试
  }
  
  // 非 200 状态码自动弹窗提示
  if (status !== 200) {
    ElMessage.error(message);
    return Promise.reject(new Error(message));
  }
  
  return res;  // res.data.data 是业务数据
}
```

**前端调用示例**：
```javascript
// 查询详情
getDetail(id).then(res => {
  const data = res.data.data;  // 直接获取业务数据
  this.form = data;
});

// 分页查询
getList(page, size, params).then(res => {
  const pageData = res.data.data;
  this.data = pageData.records;  // 数据列表
  this.page.total = pageData.total;  // 总数
});

// 提交操作
add(row).then(() => {
  this.$message.success('操作成功');
  this.onLoad(this.page);  // 刷新列表
});
```

## 四、前后端接口对照表

### 4.1 标准 CRUD 接口

| 功能 | 前端 API | 后端 URL | 方法 | 参数 | 响应 |
|------|---------|---------|------|------|------|
| 分页查询 | `getList(current, size, params)` | `/list` | GET | Query + Map | `IPage<VO>` |
| 详情查询 | `getDetail(id)` | `/detail` | GET | 实体对象 | `VO` |
| 新增 | `add(row)` | `/submit` | POST | Entity (Body) | `boolean` |
| 修改 | `update(row)` | `/submit` | POST | Entity (Body) | `boolean` |
| 删除 | `remove(ids)` | `/remove` | POST | ids (String) | `boolean` |

### 4.2 接口命名约定

| 后端 URL 后缀 | 前端方法名 | 说明 |
|--------------|-----------|------|
| `/list` | `getList` | 分页列表（推荐） |
| `/page` | `getPage` | 自定义分页（多表联查） |
| `/detail` | `getDetail` / `getXxx` | 详情查询 |
| `/info` | `getInfo` | 当前用户信息 |
| `/submit` | `add` / `update` | 新增或修改（根据有无 id 判断） |
| `/save` | `add` | 新增 |
| `/update` | `update` | 修改 |
| `/remove` | `remove` | 删除（逻辑删除） |
| `/delete` | `remove` | 删除（物理删除，少用） |
| `/enable` / `/disable` | `enable` / `disable` | 启用/禁用 |
| `/grant` | `grant` | 授权 |
| `/reset-password` | `resetPassword` | 重置密码 |

## 七、SQL 注入防护

SqlKeyword 类内置 SQL 注入防护：

**过滤的 SQL 关键词**：
```
' | % | -- | insert | delete | select | sleep | count | update | updatexml | 
group | union | drop | truncate | alter | grant | execute | exec | 
xp_cmdshell | call | declare | sql
```

**防护机制**：
1. 特殊字符过滤（`StringUtil.cleanIdentifier`）
2. 正则匹配替换 SQL 关键词
3. 二次匹配防止双写绕过
4. 发现注入尝试直接抛出 `SQLException`

## 八、调试技巧

### 8.1 后端接口调试

1. 访问 Swagger 文档：`http://localhost:18080/doc.html`
2. 查看接口定义和参数

2. 查看请求 URL、方法、参数
3. 查看响应数据结构

### 8.3 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 401 未授权 | Token 过期/无效 | 检查 Token 配置，刷新 Token |
| 403 无权限 | 权限码不匹配 | 检查 `@PreAuth` 和菜单权限配置 |
| 500 服务器错误 | 后端异常 | 查看后端日志 `tail -f logs/backend.log` |
| 数据为空 | 查询条件问题 | 检查查询参数、数据库连接 |
| 字典不显示 | 字典编码错误 | 检查 `dicUrl` 和字典配置 |
| SQL 注入报错 | 参数包含 SQL 关键词 | 检查参数名是否包含特殊字符 |
