## 2. 功能权限矩阵

### 2.1 详细权限矩阵

| 功能模块 | 功能项 | R01 | R02 | R03 | ADMIN |
|---------|--------|-----|-----|-----|-------|
| **F01 [模块 1]** | | | | | |
| | [功能项 1] | ✅ | ❌ | ❌ | ✅ |
| | [功能项 2] | ✅(自己的) | ❌ | ❌ | ✅(全部) |
| **F02 [模块 2]** | | | | | |
| | [功能项 1] | ❌ | ✅ | ❌ | ✅ |
| | [功能项 2] | ❌ | ✅ | ❌ | ✅ |
| **F03 [模块 3]** | | | | | |
| | [功能项 1] | ❌ | ❌ | ✅ | ✅ |
| **统计分析** | 查看统计 | ❌ | ✅ | ✅ | ✅ |

### 2.2 权限标识设计（权限码）

| 权限码 | 权限名称 | 说明 |
|-------|---------|------|
| `[module]:create` | 创建 [资源] | 新建 [资源] |
| `[module]:read` | 查看 [资源] | 查看 [资源] 详情 |
| `[module]:update` | 修改 [资源] | 修改 [资源] 信息 |
| `[module]:delete` | 删除 [资源] | 删除 [资源] |
| `[module]:export` | 导出 [资源] | 导出 [资源] 数据 |

### 2.3 角色权限配置

```json
{
  "R01_[角色 1]": {
    "permissions": [
      "[module]:create",
      "[module]:read",
      "[module]:update"
    ]
  },
  "R02_[角色 2]": {
    "permissions": [
      "[module]:read",
      "[module]:handle",
      "[module]:export"
    ]
  },
  "R03_[角色 3]": {
    "permissions": [
      "[module]:read",
      "[module]:approve",
      "[module]:export"
    ]
  },
  "ADMIN_系统管理员": {
    "permissions": [
      "*"
    ]
  }
}
```

## 3. 数据权限设计

### 3.1 数据权限规则

| 角色 | 数据范围 | 规则说明 |
|------|---------|---------|
| R01 [角色 1] | 个人数据 | 只能查看和操作自己创建的数据 |
| R02 [角色 2] | 本部门数据 | 可查看本部门的所有数据 |
| R03 [角色 3] | 本部门数据 | 可查看本部门的所有数据（含审核） |
| ADMIN | 全部数据 | 可查看和操作所有数据 |

### 3.2 数据权限实现

BladeX 通过 `tenant_id` 实现多租户隔离，通过数据范围配置实现更细粒度的控制。

#### 个人数据过滤

```sql
-- R01 角色
SELECT * FROM blade_[module]_[table] 
WHERE create_user = #{currentUserId}
AND tenant_id = #{currentTenantId}
```

#### 本部门数据过滤

```sql
-- R02/R03 角色
SELECT * FROM blade_[module]_[table] 
WHERE create_dept = #{currentDeptId}
AND tenant_id = #{currentTenantId}
```

#### 全部数据

```sql
-- ADMIN 角色
SELECT * FROM blade_[module]_[table] 
WHERE tenant_id = #{currentTenantId}
```

### 3.3 数据权限字段

| 表名 | 权限字段 | 说明 |
|------|---------|------|
| `blade_[module]_[table]` | `create_user` | 创建人 ID |
| `blade_[module]_[table]` | `create_dept` | 创建部门 ID |
| `blade_[module]_[table]` | `tenant_id` | 租户 ID |

## 4. 菜单权限设计

### 4.1 菜单结构

```
[系统名称]
├── [一级菜单 1]
│   ├── [二级菜单 1-1]
│   └── [二级菜单 1-2]
├── [一级菜单 2]
│   ├── [二级菜单 2-1]
│   └── [二级菜单 2-2]
└── [一级菜单 3]
```

### 4.2 菜单与按钮权限

| 菜单名称 | 菜单编号 (code) | 按钮名称 | 按钮编号 (alias) | 权限码 |
|---------|----------------|---------|-----------------|--------|
| [一级菜单] | `[module]` | - | - | `[module]` |
| └─ [子菜单] | `[module]` | 新增 | add | `[module]_add` |
| | | 修改 | edit | `[module]_edit` |
| | | 删除 | delete | `[module]_delete` |
| | | 查看 | view | `[module]_view` |
| | | 导出 | export | `[module]_export` |

> **BladeX 规范**：
> - 菜单编号 (`code`) 与 Controller 的 `@PreAuth(menu = "xxx")` 对应
> - 按钮权限码前端通过 `permission.xxx` 控制显示/隐藏
> - 菜单权限通过 `blade_menu` 表配置

| [菜单 1] | ✅ | ✅ | ✅ | ✅ |
| [菜单 2] | ❌ | ✅ | ✅ | ✅ |
| [菜单 3] | ❌ | ❌ | ✅ | ✅ |

## 5. 特殊权限规则

### 5.1 同部门可查看规则

| 规则 | 说明 |
|------|------|
| 适用场景 | 列表查询时 |
| 规则描述 | 同一部门的人员可查看本部门的所有数据 |
| 实现方式 | 查询条件：`create_dept = 当前用户部门 ID` |
| 注意事项 | 只能查看，不能修改他人创建的数据 |

### 5.2 仅创建人可维护规则

| 规则 | 说明 |
|------|------|
| 适用场景 | 修改/删除操作时 |
| 规则描述 | 仅创建人可修改/删除自己创建的数据 |
| 例外 | 系统管理员可操作所有数据 |

### 5.3 审核权限规则

| 规则 | 说明 |
|------|------|
| 适用场景 | 审核操作时 |
| 权限条件 | 具有审核角色的用户 |
| 数据范围 | 本部门提交的数据 |

## 6. 权限校验实现

### 6.1 前端权限控制

**菜单权限**：

```javascript
// router/index.js
{
  path: '/module',
  component: Layout,
  meta: {
    roles: ['R02', 'R03', 'ADMIN'] // 可访问的角色
  }
}
```

**按钮权限**：

```vue
<template>
  <el-button 
    v-if="permission['module_add']" 
    type="primary" 
    @click="handleAdd"
  >
    新增
  </el-button>
</template>
```

### 6.2 后端权限控制

**接口权限注解**：

```java
// Controller 方法
@PostMapping("/save")
@PreAuth(menu = "module_add")  // 菜单权限
public R<Boolean> save(@RequestBody Entity entity) {
    // 业务逻辑
}

@DeleteMapping("/remove")
@PreAuth(menu = "module_delete")  // 删除权限
public R<Boolean> remove(@RequestParam String ids) {
    // 业务逻辑
}
```

**数据权限 AOP**：

```java
// Service 层
@DataScope(deptAlias = "create_dept", userAlias = "create_user")
public List<Entity> list(Query query) {
    return baseMapper.list(query);
}
```

## 7. 权限管理流程

### 7.1 角色分配流程

```
┌─────────────────────────────────────────────────────────────┐
│                    角色分配流程                              │
│                                                             │
│   ┌────────────────┐                                        │
│   │ 系统用户创建   │ ← BladeX 用户管理模块                  │
│   └───────┬────────┘                                        │
│           │                                                 │
│           ▼                                                 │
│   ┌────────────────┐                                        │
│   │ 分配角色       │ ← 用户角色绑定                         │
│   │ (一个用户可多角) │                                       │
│   └───────┬────────┘                                        │
│           │                                                 │
│           ▼                                                 │
│   ┌────────────────┐                                        │
│   │ 权限生效       │ ← 重新登录或刷新 Token                  │
│   └────────────────┘                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 权限变更流程

| 变更类型 | 操作方式 | 影响范围 |
|---------|---------|---------|
| 新增用户 | 用户管理 - 新建 | 获得分配的角色权限 |
| 分配角色 | 用户管理 - 编辑 | 获得新角色权限 |
| 取消角色 | 用户管理 - 编辑 | 取消对应权限 |
| 删除用户 | 用户管理 - 删除 | 取消所有权限 |

## 8. 权限审计

### 8.1 审计日志记录

| 审计项 | 记录内容 |
|--------|---------|
| 登录日志 | 用户 ID、登录时间、登录 IP、登录设备 |
| 操作日志 | 用户 ID、操作时间、操作类型、操作对象、操作结果 |
| 权限变更日志 | 变更人、变更时间、变更内容、被变更人 |

BladeX 自动记录操作日志，通过 `blade_log` 表存储。

### 8.2 敏感操作审计

| 操作 | 审计级别 | 说明 |
|------|---------|------|
| 删除数据 | 高 | 记录删除人、删除内容、删除时间 |
| 导出数据 | 中 | 记录导出人、导出内容、导出时间 |
| 权限变更 | 高 | 记录变更人、变更内容、被变更人 |
| 审核操作 | 高 | 记录审核人、审核时间、审核意见 |

## 9. BladeX 权限配置 SQL 示例

### 9.1 角色配置

```sql
-- 新增角色
INSERT INTO blade_role (id, role_name, role_code, role_alias, sort, is_deleted)
VALUES (序列，'角色名称', 'role_code', '角色别名', 1, 0);
```

### 9.2 菜单权限配置

```sql
-- 查询现有菜单
SELECT * FROM blade_menu WHERE parent_id = 0;

-- 新增一级菜单
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, component, sort, is_deleted)
VALUES (序列，0, 'module', '模块名称', 'module', '/module', 'views/module/index', 1, 0);

-- 新增二级菜单
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, component, sort, is_deleted)
VALUES (序列，父级菜单 ID, 'module', '子菜单名称', 'module_list', '/module/list', 'views/module/list', 1, 0);
```

### 9.3 角色菜单关联

```sql
-- 角色绑定菜单
INSERT INTO blade_role_menu (menu_id, role_id)
SELECT 菜单 ID, 角色 ID FROM DUAL;
```
