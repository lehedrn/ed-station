## 补充说明

### 文档约定

1. **命名规范**：表名、字段名遵循 BladeX 规范，使用下划线命名法
2. **状态编码**：采用拼音首字母缩写形式，如 DSL（待受理）、YSL（已受理）
3. **流程图**：使用 Mermaid 语法绘制，支持在 Markdown 中渲染
4. **接口规范**：RESTful 风格，统一响应格式 `R<T>`

### BladeX 框架规范

- **多租户隔离**：所有业务表包含 `tenant_id` 字段
- **审计字段**：包含 `create_user`、`create_time`、`update_user`、`update_time`
- **逻辑删除**：使用 `is_deleted` 字段代替物理删除
- **字典管理**：通过 `blade_dict` 表统一管理数据字典
- **权限控制**：基于 `blade_role`、`blade_menu` 实现 RBAC 权限
- **接口响应**：统一使用 `R<T>` 包装类

### 二期规划

在总文档中明确标注一期暂不实现的功能，便于后续迭代规划。
