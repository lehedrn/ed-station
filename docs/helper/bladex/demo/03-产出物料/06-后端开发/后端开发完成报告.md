## 业务逻辑实现

### 1. 实施编码自动生成
- 格式：`AFFAIR` + `yyyyMMddHHmmss` + `6 位序号`
- 示例：`AFFAIR20260402103000000001`

### 2. 时限校验
- 承诺时限不能大于法定时限
- 违反时抛出 `IllegalArgumentException`

### 3. 创建人权限校验
- 仅创建人可编辑/删除自己创建的事项
- 发布/下架操作也仅限创建人

### 4. 材料子表处理
- 新增时同步保存材料列表
- 修改时先删除旧材料，再新增新材料
- 删除事项时同步逻辑删除关联材料

### 5. 发布/下架逻辑
- 发布：状态设为 1（正常），记录发布时间
- 下架：状态设为 2（下架）
