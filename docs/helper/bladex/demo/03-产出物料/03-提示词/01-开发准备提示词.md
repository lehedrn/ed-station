# 阶段 1：开发准备提示词

> 用于开发前分析、数据库设计、字典与权限配置

---

## 使用说明

本提示词文档用于指导完成开发准备阶段的三项核心任务：
1. **开发前分析** - 明确模块基本信息、原型设计
2. **数据库设计** - 生成建表 SQL 脚本
3. **字典与权限配置** - 生成字典和菜单 SQL 脚本

---

## 提示词 1-1：开发前分析

**使用场景**：开始新模块开发前，需要明确模块基本信息和原型设计

**复制以下内容给 Claude**：

```
我现在需要开发一个新的业务模块，请帮我进行开发前分析。

## 项目背景

我正在基于 BladeX 4.8.0 框架开发"政务服务事项管理系统"。

## 需求文档

请先阅读以下需求文档：
- 需求总文档：{需求文档路径}/01-需求总文档.md
- 功能说明子文档：{需求文档路径}/02-功能说明子文档.md
- 界面操作子文档：{需求文档路径}/03-界面操作子文档.md

## 任务要求

请帮我分析并输出以下信息：

### 1. 模块基本信息

请按照以下格式输出模块基本信息表：

| 项目 | 值 | 说明 |
|------|-----|------|
| 模块名称 | [中文名称] | 用于菜单显示 |
| 模块代码 | [英文标识] | 用于权限码、路由等 |
| 所属模块目录 | [desk/system/auth 等] | 决定后端包路径和前端路由前缀 |
| 数据库表名 | [表名列表] | 遵循 BladeX 命名规范 |
| 实体类名 | [类名列表] | Java 类名（大驼峰） |
| 前端路由 | [路由路径] | 浏览器 URL 路径 |

### 2. 原型页面设计

请根据界面操作子文档，输出以下原型设计：

#### 2.1 列表页面草图
- 显示字段清单
- 搜索条件清单
- 工具栏按钮清单

#### 2.2 表单页面草图
- 表单字段清单（包含字段类型、是否必填、组件类型）
- 表单布局建议

#### 2.3 字段属性清单
请以表格形式列出所有字段：
| 字段 | 类型 | 必填 | 搜索 | 字典 | 组件 |

### 3. 技术栈确认

确认本模块使用的技术栈：
- 后端：Spring Boot 3.2.10 + BladeX 4.8.0 + MyBatis-Plus
- 前端：Vue 3 + Vite + Element Plus + Avue 3.7

## 输出要求

1. 使用 Markdown 格式输出
2. 表格清晰易读
3. 原型草图使用 ASCII 艺术形式
4. 字段清单完整无遗漏
```

## 提示词 1-2：数据库设计

**使用场景**：完成开发前分析后，需要设计数据库表结构

**复制以下内容给 Claude**：

```
我已经完成了开发前分析，现在需要设计数据库表结构。

## 项目信息

- 模块名称：{模块名称}
- 模块代码：{模块代码}
- 主表名称：{数据库表名}

## 需求文档

- 数据接口子文档：{需求文档路径}/06-数据接口子文档.md
- 业务流程子文档：{需求文档路径}/05-业务流程和规则子文档.md

## 任务要求

请帮我设计完整的数据库表结构，并生成 SQL 建表脚本。

### 1. 数据库设计规范

请遵循 BladeX 框架规范：

**主表必须包含的字段**：
| 字段名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| id | bigint | 主键（雪花算法） | - |
| tenant_id | varchar(12) | 租户 ID | '000000' |
| create_user | bigint | 创建人 | NULL |
| create_dept | bigint | 创建部门 | NULL |
| create_time | datetime | 创建时间 | NULL |
| update_user | bigint | 修改人 | NULL |
| update_time | datetime | 修改时间 | NULL |
| status | int | 状态 | 1 |
| is_deleted | int | 是否删除 | 0 |

### 2. 业务字段设计

根据需求文档中的数据库设计，列出所有业务字段：
- 字段名（下划线命名）
- 字段类型
- 字段说明
- 是否需要字典

### 3. 索引设计

设计合理的索引：
- 主键索引（自动）
- 外键索引（如有）
- 查询条件索引（根据搜索字段）
- 唯一索引（如有唯一性要求）

### 4. 输出 SQL 建表脚本

请生成完整的 SQL 建表脚本，包含：
- 文件头注释（模块名、表名、创建时间）
- DROP TABLE IF EXISTS 语句
- CREATE TABLE 语句（包含所有字段和注释）
- 索引创建语句
- 表注释

SQL 脚本格式参考：
```sql
-- =============================================
-- 模块：{模块名称}
-- 表名：{数据库表名}
-- 创建时间：{当前日期}
-- =============================================

DROP TABLE IF EXISTS `{表名}`;
CREATE TABLE `{表名}`  (
  `id` bigint NOT NULL COMMENT '主键',
  -- ... 其他字段
  PRIMARY KEY (`id`)
) ENGINE = InnoDB CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci COMMENT = '{表注释}';
```

### 5. 子表设计（如有）

如果有子表（如材料关联表），请同样设计子表结构。

## 输出要求

1. SQL 脚本可直接执行
2. 字段注释完整准确
3. 遵循 BladeX 命名规范
4. 考虑查询性能的索引设计
```

## 提示词 1-3：字典与权限配置

**使用场景**：数据库设计完成后，需要配置字典和菜单权限

**复制以下内容给 Claude**：

```
数据库设计已完成，现在需要生成字典配置和权限配置的 SQL 脚本。

## 项目信息

- 模块名称：{模块名称}
- 模块代码：{模块代码}
- 所属模块目录：{所属模块目录}（如 desk/system/auth）
- 需求文档路径：{需求文档路径}

## 需求文档

- 公共字典子文档：{需求文档路径}/07-公共字典子文档.md
- 角色权限子文档：{需求文档路径}/04-角色权限子文档.md

## 任务要求

### 1. 字典配置 SQL

#### 1.1 分析字典需求

请根据需求文档分析：
- 需要哪些字典（字典编码、字典名称）
- 每个字典包含哪些字典项（dict_key、dict_value）

#### 1.2 生成字典 SQL 脚本

请生成完整的字典 SQL 脚本，包含：
- 字典类型插入（顶级节点，dict_key = '-1'）
- 字典数据插入（子节点，dict_key 为实际业务值）

SQL 脚本格式：
```sql
-- =============================================
-- 字典：{字典名称}
-- 字典编码：{字典编码}
-- 所属模块：{模块名称}
-- =============================================

-- 1. 插入字典类型（父级，dict_key = -1）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES ({ID}, 0, '{字典编码}', '-1', '{字典名称}', 1, NULL, 0, 1, 0);

-- 2. 插入字典值（子级）
INSERT INTO blade_dict (id, parent_id, code, dict_key, dict_value, sort, remark, is_sealed, status, is_deleted) 
VALUES 
({ID}, {父级 ID}, '{字典编码}', '{业务值}', '{显示名称}', {排序}, NULL, 0, 1, 0);
```

### 2. 权限配置 SQL

#### 2.1 分析权限需求

请根据需求文档分析：
- 需要哪些菜单（一级菜单、按钮菜单）
- 每个菜单的权限码（code 字段）
- 菜单的层级关系

#### 2.2 生成菜单 SQL 脚本

请生成完整的菜单 SQL 脚本，包含：
- 一级菜单（category=1）
- 按钮菜单（category=2）

菜单层级关系：
```
顶级菜单 (parent_id = 0)
  └── 一级菜单 (parent_id = 顶级菜单 ID)
      └── 按钮/功能菜单 (parent_id = 一级菜单 ID)
```

按钮类型说明：
- action=1：工具栏按钮（如新增、删除）
- action=2：操作栏按钮（如编辑、查看）
- action=3：工具操作栏按钮

SQL 脚本格式：
```sql
-- =============================================
-- 菜单：{模块名称}
-- 所属模块：{所属模块目录}
-- =============================================

-- 1. 插入一级菜单
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, source, sort, category, action, is_open, component, remark, is_deleted)
VALUES 
({ID}, {父级 ID}, '{模块代码}', '{模块名称}', 'menu', '/{路由前缀}', 'iconfont icon-icon_name', {排序}, 1, 0, 1, '', NULL, 0);

-- 2. 插入按钮菜单
INSERT INTO blade_menu (id, parent_id, code, name, alias, path, source, sort, category, action, is_open, component, remark, is_deleted)
VALUES 
-- 新增按钮
({ID}, {一级菜单 ID}, '{模块代码}_add', '新增', 'add', '/{路由前缀}/add', 'plus', 1, 2, 1, 1, '', NULL, 0),
-- 修改按钮
({ID}, {一级菜单 ID}, '{模块代码}_edit', '修改', 'edit', '/{路由前缀}/edit', 'form', 2, 2, 2, 1, '', NULL, 0),
-- 删除按钮
({ID}, {一级菜单 ID}, '{模块代码}_delete', '删除', 'delete', '/api/blade-{模块目录}/{模块代码}/remove', 'delete', 3, 2, 3, 1, '', NULL, 0),
-- 查看按钮
({ID}, {一级菜单 ID}, '{模块代码}_view', '查看', 'view', '/{路由前缀}/view', 'file-text', 4, 2, 2, 1, '', NULL, 0);
```

### 3. 权限码对照表

请输出权限码对照表：
| 权限码 | 说明 | Controller 注解 |
|--------|------|----------------|
| {模块代码} | 菜单访问权限 | @PreAuth(menu = "{模块代码}") |
| {模块代码}_add | 新增权限 | - |
| {模块代码}_edit | 修改权限 | - |
| {模块代码}_delete | 删除权限 | - |
| {模块代码}_view | 查看权限 | - |

### 4. 所属模块 ID 确认

请查找并确认所属模块的父级 ID：
- 如果是 desk（工作台），ID = 1123598815738675201
- 如果是 system（系统管理），ID = 1123598815738675202
- 如果是 auth（认证授权），ID = 1123598815738675203

## 输出要求

1. SQL 脚本可直接执行
2. ID 使用合理的雪花算法生成规则
3. 菜单 code 与权限码一致
4. 字典编码与后端枚举一致

## 补充说明

BladeX 字典表结构：
- code: 字典编码
- dict_key: 字典键值（顶级节点使用 -1）
- dict_value: 字典显示名称
- parent_id: 父主键（顶级为 0，子节点指向父级 id）

BladeX 菜单表结构：
- id: 主键（雪花算法）
- parent_id: 父主键
- code: 菜单编号（权限码）
- name: 菜单名称
- category: 1=一级菜单，2=按钮菜单
- action: 1=工具栏，2=操作栏，3=工具操作栏
```

1. 先执行 **提示词 1-1**，完成开发前分析
2. 基于分析结果执行 **提示词 1-2**，生成数据库设计
3. 执行 SQL 脚本创建数据库表
4. 执行 **提示词 1-3**，生成字典和权限配置 SQL
5. 执行字典和权限 SQL 脚本
6. 验证数据库和配置是否正确
