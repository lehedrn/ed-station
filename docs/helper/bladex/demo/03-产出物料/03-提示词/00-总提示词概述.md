## 提示词使用说明

### 基本使用流程

1. **按顺序执行**：按照阶段 1 → 阶段 2 → 阶段 3A/3B → 阶段 4 的顺序执行
2. **准备上下文**：每次使用提示词前，确保已准备好相关的需求文档
3. **复制提示词**：将对应阶段的提示词复制到与 Claude 的对话中
4. **提供补充信息**：根据提示词要求，提供必要的项目路径、文件等信息

### 提示词中的占位符

提示词中包含以下占位符，使用时需要替换为实际内容：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{模块名称}` | 中文模块名称 | `政务服务事项管理` |
| `{模块代码}` | 英文模块标识 | `affair` |
| `{所属模块目录}` | BladeX 模块分类 | `system` |
| `{数据库表名}` | 主表名称 | `blade_affair` |
| `{前端路由前缀}` | 前端路由路径 | `/affair` |
| `{需求文档路径}` | 细化需求文档所在路径 | `docs/helper/bladex/demo/03-产出物料/02-细化需求文档/` |

### 与 Claude 协作的最佳实践

1. **一次一个任务**：每个提示词聚焦一个阶段的任务，不要跳跃
2. **保存中间结果**：将 Claude 生成的 SQL、代码等保存到对应文件
3. **及时验证**：每完成一个阶段，及时验证结果是否符合预期
4. **上下文管理**：长对话时，适时开启新对话避免上下文超限
