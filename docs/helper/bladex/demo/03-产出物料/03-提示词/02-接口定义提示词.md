# 阶段 2：接口定义提示词

> 用于接口契约定义、前后端确认

---

## 使用说明

本提示词文档用于指导完成接口定义阶段的任务：
1. **接口列表定义** - 基于需求和权限配置定义完整接口清单
2. **接口详细设计** - 定义每个接口的请求参数和响应结构
3. **前后端确认** - 生成接口契约文档供前后端确认

> **为什么这个阶段很重要？**
> - 接口契约是前后端并行开发的基础
> - 避免后期因接口不一致导致的返工
> - 前端可以根据接口定义配置 Avue column
> - 后端可以根据接口定义编写 Controller 框架

### 2. 接口详细设计

#### 2.1 分页查询接口

请定义详细的请求参数和响应结构：

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| current | int | 是 | 当前页码（从 1 开始） | 1 |
| size | int | 是 | 每页条数 | 10 |
| affairName | string | 否 | 事项名称（模糊查询） | 测试 |
| affairType | string | 否 | 事项类别（精确匹配） | 01 |
| status | int | 否 | 事项状态（1-正常 2-下架） | 1 |

**响应结构**：
```json
{
  "code": 200,
  "success": true,
  "msg": "查询成功",
  "data": {
    "records": [
      {
        "id": "1234567890123456789",
        "affairName": "测试事项 1",
        "affairType": "01",
        "affairTypeDict": "行政许可",
        "status": 1,
        "statusDict": "正常"
      }
    ],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  }
}
```

1. 接口列表完整，覆盖所有功能
2. 请求参数和响应结构详细

## 补充说明

BladeX 接口规范：
- 基础路径：`/blade-{模块目录}/{模块代码}`
- 统一响应格式：`R<T>`，包含 code、success、msg、data
- 分页响应：包含 records、total、size、current、pages

### 3. 生成前端 API 封装框架

请生成前端 API 封装的框架代码（JavaScript）：

```javascript
// src/api/{模块目录}/{模块代码}.js
import request from '@/axios';

/**
 * 分页查询
 */
export const getList = (current, size, params) => {
  return request({
    url: '/blade-{模块目录}/{模块代码}/list',
    method: 'get',
    params: { ...params, current, size },
    cryptoToken: false,
    cryptoData: false,
  });
};

/**
 * 详情查询
 */
export const getDetail = (id) => {
  return request({
    url: '/blade-{模块目录}/{模块代码}/detail',
    method: 'get',
    params: { id },
    cryptoToken: false,
    cryptoData: false,
  });
};

// ... 其他接口封装
```
