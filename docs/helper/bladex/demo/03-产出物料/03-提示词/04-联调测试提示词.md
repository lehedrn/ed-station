### 2. 常见问题排查

请提供常见问题的排查方法：

#### 2.1 接口 404

**可能原因**：
- 后端 Controller 路径配置错误
- 前端 API 路径配置错误
- 代理配置错误

**排查步骤**：
1. 检查 Controller 的@RequestMapping 路径
2. 检查前端 API 的 url 配置
3. 检查 vite.config.mjs 的 proxy 配置

#### 2.2 接口 401/403

**可能原因**：
- Token 过期或无效
- 权限配置未完成

**排查步骤**：
1. 检查登录状态
2. 检查菜单权限 SQL 是否执行
3. 检查角色是否绑定菜单

#### 2.3 数据为空

**可能原因**：
- 数据库无数据
- 查询条件问题
- 字典配置问题

**排查步骤**：
1. 直接查询数据库确认数据
2. 检查 SQL 日志
3. 检查字典数据是否加载

### 4. Network 面板检查

请提供 Network 面板检查要点：

1. **请求 URL**：确认请求地址正确
2. **请求方法**：确认 GET/POST 方法正确
3. **请求头**：确认 Authorization 包含 Token
4. **请求参数**：确认参数格式和值正确
5. **响应状态码**：200 表示成功
6. **响应数据**：确认 data 结构符合预期

### 3. 权限问题排查

请提供权限问题的排查方法：

#### 菜单不显示

**排查步骤**：
1. 检查 blade_menu 表是否有菜单配置
2. 检查 blade_role_menu 表是否绑定角色
3. 检查用户是否绑定对应角色
4. 清除缓存后重新登录

#### 按钮不显示

**排查步骤**：
1. 检查 blade_menu 表是否有按钮配置
2. 检查按钮的父级菜单是否正确
3. 检查权限码是否与前端配置一致
4. 检查前端 permission 对象是否包含权限码

#### 数据权限失效

**排查步骤**：
1. 检查 Service 层是否有权限校验代码
2. 检查 createUser 字段是否正确
3. 检查当前用户 ID 获取是否正确
