### 中文参数 URL 编码

基于经验教训（2026-04-02），cURL 测试中中文参数必须进行 URL 编码，否则会导致 400 错误。

**解决方案**：使用 Python 的 `urllib.parse.quote()` 函数进行编码：

```bash
# 使用 Python 进行 URL 编码
AFFAIR_NAME_ENCODED=$(python3 -c "import urllib.parse; print(urllib.parse.quote('测试', encoding='utf-8'))")
RESPONSE=$(curl -s -X GET "$BASE_URL/blade-affair/affair/list?current=1&size=10&affairName=${AFFAIR_NAME_ENCODED}" ...)
```

**注意**：在 WSL2 环境下，中文字符处理可能不一致，建议始终使用 URL 编码。

## ⚠️ 重要注意事项

### 选择器严格模式

基于经验教训（2026-04-02），Playwright 的严格模式不允许选择器匹配多个元素。

**问题示例**：
```javascript
// ❌ 错误：匹配多个元素
const crudContainer = page.locator('.avue-crud, .basic-container');
// 报错：strict mode violation: locator resolved to 2 elements
```

**解决方案**：使用更精确的选择器

```javascript
// ✅ 正确：使用更精确的选择器
const crudContainer = page.locator('.avue-crud.avue-crud--card');

// 或使用 getByText/GetByRole 等定位方式
const crudContainer = page.getByText('事项名称:').first();

// 或使用 data-testid
const crudContainer = page.getByTestId('affair-crud');
```

### 测试数据准备

基于经验教训（2026-04-02），数据依赖的测试（T03/T04/T05/T06）需要预先准备测试数据。

**解决方案**：在测试前通过 API 创建测试事项

```javascript
// prepare-test-data.js
const { request } = require('@playwright/test');

async function prepareTestData() {
  const apiContext = await request.newContext({
    baseURL: 'http://localhost:18080',
  });
  
  // 1. 登录获取 Token
  const tokenResponse = await apiContext.post('/api/auth/login', {
    data: {
      tenantId: '000000',
      username: 'admin',
      password: 'admin',
    },
  });
  
  // 2. 创建测试事项
  const timestamp = Date.now();
  const createResponse = await apiContext.post('/blade-affair/affair/save', {
    headers: { Authorization: `Bearer ${token}` },
    data: {
      affairName: `测试事项_${timestamp}`,
      // ... 其他字段
    },
  });
  
  // 3. 通过列表查询获取生成的 ID
  const listResponse = await apiContext.get(
    `/blade-affair/affair/list?current=1&size=1&affairName=测试事项_${timestamp}`
  );
  const testData = await listResponse.json();
  const testId = testData.data.records[0].id;
  
  return { testId, timestamp };
}
```

## 自动化测试特点

### cURL 接口测试
- **优势**: 快速、轻量、易于调试
- **适用**: 后端接口功能验证、边界条件测试
- **输出**: 命令行测试结果、日志文件

### Playwright UI 测试
- **优势**: 真实用户场景、可视化验证、截图证明
- **适用**: 前端功能验证、用户体验测试
- **输出**: HTML 报告、测试截图

### 两者结合
- **覆盖率**: 接口 + 界面，全链路覆盖
- **效率**: 可并行执行，快速反馈
- **可靠性**: 自动化执行，避免人为因素
