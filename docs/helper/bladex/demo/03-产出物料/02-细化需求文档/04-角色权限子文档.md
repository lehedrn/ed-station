## 1. 角色体系设计

### 1.1 角色定义

| 角色 ID | 角色名称 | 角色编码 | 角色描述 | 用户群体 |
|--------|---------|---------|---------|---------|
| R01 | 事项管理员 | affair_admin | 负责事项的编制、录入、维护、发布/下架 | 政务服务中心工作人员 |
| ADMIN | 系统管理员 | administrator | 管理基础配置、用户、角色 | IT 运维人员 |
| VIEWER | 普通用户 | viewer | 查看已发布事项（预留） | 公众用户 |

**注**：根据 2-A 确认，本系统简化版不实现审核流程，审核人员角色已移除。

## 3. 数据权限设计

### 3.1 数据权限规则

| 角色 | 数据范围 | 规则说明 |
|------|---------|---------|
| R01 事项管理员 | 个人数据 | 只能查看和操作自己创建的数据 |
| ADMIN 系统管理员 | 全部数据 | 可查看和操作所有数据 |
| VIEWER 普通用户 | 已发布数据 | 仅可查看状态为正常的事项 |

### 3.2 数据权限实现

#### 个人数据过滤

```sql
-- R01 角色（事项管理员）
SELECT * FROM blade_affair 
WHERE create_user = #{currentUserId}
AND is_deleted = 0
```

#### 全部数据

```sql
-- ADMIN 角色（系统管理员）
SELECT * FROM blade_affair 
WHERE is_deleted = 0
```

#### 已发布数据

```sql
-- VIEWER 角色（普通用户）
SELECT * FROM blade_affair 
WHERE status = 1
AND is_deleted = 0
```

### 3.3 数据权限字段

| 表名 | 权限字段 | 说明 |
|------|---------|------|
| `blade_affair` | `create_user` | 创建人 ID |
| `blade_affair` | `status` | 事项状态（1-正常，2-下架） |

|---------|----------------|---------|-----------------|--------|

## 5. 特殊权限规则

### 5.1 仅创建人可维护规则

| 规则 | 说明 |
|------|------|
| 适用场景 | 修改/删除操作时 |
| 规则描述 | 仅创建人可修改/删除自己创建的数据 |
| 例外 | 系统管理员可操作所有数据 |
| 实现方式 | 后端校验 `create_user == currentUserId` |

### 5.2 敏感操作审计

| 操作 | 审计级别 | 说明 |
|------|---------|------|
| 删除数据 | 高 | 记录删除人、删除内容、删除时间 |
| 发布事项 | 中 | 记录发布人、发布时间 |
| 下架事项 | 中 | 记录下架人、下架时间 |

## 6. 权限校验实现

### 6.1 前端权限控制

**菜单权限**：

```javascript
// router/index.js
{
  path: '/affair',
  component: Layout,
  meta: {
    roles: ['affair_admin', 'administrator'] // 可访问的角色
  }
}
```

**按钮权限**：

```vue
<template>
  <el-button 
    v-if="permission['affair_manage_add']" 
    type="primary" 
    @click="handleAdd"
  >
    新增
  </el-button>
  
  <el-button 
    v-if="permission['affair_manage_edit']" 
    type="primary" 
    @click="handleEdit"
  >
    修改
  </el-button>
</template>
```

### 6.2 后端权限控制

**接口权限注解**：

```java
// Controller 方法
@PostMapping("/save")
@PreAuth(menu = "affair_manage_add")
public R<Boolean> save(@RequestBody AffairEntity entity) {
    // 业务逻辑
}

@DeleteMapping("/remove")
@PreAuth(menu = "affair_manage_delete")
public R<Boolean> remove(@RequestParam String ids) {
    // 业务逻辑
}
```

**数据权限校验**：

```java
// Service 层
@Override
public boolean update(AffairEntity entity) {
    // 校验是否为创建人
    AffairEntity original = getById(entity.getId());
    if (!original.getCreateUser().equals(getCurrentUserId())) {
        throw new BusinessException("无权限修改他人数据");
    }
    return super.updateById(entity);
}
```
