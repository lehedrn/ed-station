## 1. 界面设计总则

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| 一致性 | 同类操作使用一致的交互方式 |
| 易用性 | 符合用户操作习惯，减少不必要的点击 |
| 反馈性 | 操作后及时给予明确反馈 |
| 容错性 | 关键操作需确认，支持撤销 |
| 效率性 | 常用功能便捷可达 |

### 1.2 技术规范

| 端 | UI 框架 | 组件库 |
|---|--------|--------|
| PC 端 | Vue 3 (Options API) | Element Plus + Avue 3.7 |

### 1.3 响应式设计

| 端 | 最小宽度 | 推荐分辨率 |
|---|---------|-----------|
| PC 端 | 1280px | 1920×1080 |

## 4. 交互规范

### 4.1 确认类操作

| 操作 | 确认文案 |
|------|---------|
| 保存 | "确认保存吗？" |
| 删除 | "确认删除该记录吗？删除后无法恢复。" |
| 批量删除 | "确认删除选中的 X 条记录吗？" |

### 4.2 成功/失败提示

| 操作 | 成功提示 | 失败提示 |
|------|---------|---------|
| 保存 | "保存成功" | "保存失败：{原因}" |
| 删除 | "删除成功" | "删除失败：{原因}" |
| 发布 | "发布成功" | "发布失败：{原因}" |
| 下架 | "下架成功" | "下架失败：{原因}" |

使用 Element Plus 的 `ElMessage` 组件：
- 成功：`ElMessage.success('操作成功')`
- 失败：`ElMessage.error('操作失败：' + message)`

### 4.3 加载状态

| 场景 | 加载方式 |
|------|---------|
| 列表加载 | 表格 loading 遮罩 |
| 详情加载 | 页面骨架屏 |
| 提交/保存 | 按钮 loading 状态 |
| 文件上传 | 进度条显示 |

## 5. BladeX 前端规范

### 5.1 API 请求封装

所有接口请求使用统一的 axios 封装：

```javascript
import request from '@/axios';

// 列表示例
export const getList = (current, size, params) => {
  return request({
    url: '/blade-affair/affair/list',
    method: 'get',
    params: { ...params, current, size },
    cryptoToken: false,
    cryptoData: false,
  });
};
```

### 5.2 字典使用

字典数据通过 BladeX 统一字典接口获取：

```javascript
// Avue 配置示例
{
  label: '事项类别',
  type: 'select',
  dicUrl: '/blade-system/dict/dictionary?code=affair_type',
  props: { label: 'dictValue', value: 'dictKey' },
  dataType: 'string',
  prop: 'affair_type',
  rules: [{ required: true, message: '请选择事项类别', trigger: 'change' }]
}
```

### 5.3 权限控制

按钮权限通过 `permission` 对象控制：

```vue
<el-button 
  v-if="permission['affair_manage_add']" 
  type="primary" 
  @click="handleAdd"
>
  新增
</el-button>
```

### 5.4 日期格式

- 列表展示：`YYYY-MM-DD HH:mm:ss`
- 日期选择：`YYYY-MM-DD`
- 时间选择：`HH:mm:ss`

使用 Dayjs 进行日期格式化。

### 5.5 文件上传

```vue
<el-upload
  action="/blade-resource/attach/upload"
  :before-upload="handleBeforeUpload"
  :on-success="handleSuccess"
  :accept="'.pdf,.doc,.docx,.jpg,.png'"
>
  <el-button type="primary">选择文件</el-button>
  <template #tip>
    <div class="el-upload__tip">
      支持 pdf/doc/docx/jpg/png 格式，单文件≤20MB
    </div>
  </template>
</el-upload>
```

## 6. 特殊场景处理

### 6.1 表单编辑权限

- 仅创建人可编辑自己创建的事项
- 系统管理员可编辑所有事项
- 通过后端权限校验 + 前端按钮隐藏双重控制

### 6.2 材料子表操作

- 支持动态添加/删除材料项
- 每项材料独立上传附件
- 支持拖拽排序

### 6.3 空状态展示

列表无数据时，显示空状态组件：

```vue
<el-empty description="暂无事项数据" />
```
