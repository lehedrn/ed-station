## 1. 界面设计总则

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| 一致性 | 同类操作使用一致的交互方式 |
| 易用性 | 符合用户操作习惯，减少不必要的点击 |
| 反馈性 | 操作后及时给予明确反馈 |
| 容错性 | 关键操作需确认，支持撤销 |
| 效率性 | 常用功能便捷可达 |

### 1.2 技术规范

| 端 | UI 框架 | 组件库 |
|---|--------|--------|
| PC 端 | Vue 3 (Options API) | Element Plus + Avue 3.7 |

### 1.3 响应式设计

| 端 | 最小宽度 | 推荐分辨率 |
|---|---------|-----------|
| PC 端 | 1280px | 1920×1080 |

## 5. BladeX 前端规范

### 5.1 API 请求封装

所有接口请求使用统一的 axios 封装：

```javascript
import request from '@/axios';

// 列表示例
export const getList = (current, size, params) => {
  return request({
    url: '/blade-affair/affair/list',
    method: 'get',
    params: { ...params, current, size },
    cryptoToken: false,
    cryptoData: false,
  });
};
```

### 5.2 字典使用

字典数据通过 BladeX 统一字典接口获取：

```javascript
// Avue 配置示例
{
  label: '事项类别',
  type: 'select',
  dicUrl: '/blade-system/dict/dictionary?code=affair_type',
  props: { label: 'dictValue', value: 'dictKey' },
  dataType: 'string',
  prop: 'affair_type',
  rules: [{ required: true, message: '请选择事项类别', trigger: 'change' }]
}
```

### 5.3 权限控制

按钮权限通过 `permission` 对象控制：

```vue
<el-button 
  v-if="permission['affair_manage_add']" 
  type="primary" 
  @click="handleAdd"
>
  新增
</el-button>
```

### 5.4 日期格式

- 列表展示：`YYYY-MM-DD HH:mm:ss`
- 日期选择：`YYYY-MM-DD`
- 时间选择：`HH:mm:ss`

使用 Dayjs 进行日期格式化。

### 5.5 文件上传

```vue
<el-upload
  action="/blade-resource/attach/upload"
  :before-upload="handleBeforeUpload"
  :on-success="handleSuccess"
  :accept="'.pdf,.doc,.docx,.jpg,.png'"
>
  <el-button type="primary">选择文件</el-button>
  <template #tip>
    <div class="el-upload__tip">
      支持 pdf/doc/docx/jpg/png 格式，单文件≤20MB
    </div>
  </template>
</el-upload>
```

## 6. 特殊场景处理

### 6.1 表单编辑权限

- 仅创建人可编辑自己创建的事项
- 系统管理员可编辑所有事项
- 通过后端权限校验 + 前端按钮隐藏双重控制

### 6.2 材料子表操作

- 支持动态添加/删除材料项
- 每项材料独立上传附件
- 支持拖拽排序

### 6.3 空状态展示

列表无数据时，显示空状态组件：

```vue
<el-empty description="暂无事项数据" />
```

## 8. 界面响应式设计

### 8.1 表单响应式规则

| 屏幕宽度 | 表单布局 |
|---------|---------|
| ≥1920px | 双列布局 |
| ≥1280px | 单列布局 |
| <1280px | 单列布局（推荐最小宽度） |

### 8.2 表格响应式规则

| 屏幕宽度 | 表格处理 |
|---------|---------|
| ≥1920px | 显示全部列 |
| ≥1280px | 隐藏非关键列（简称、备注等） |
| <1280px | 显示横向滚动条 |

### 8.3 弹窗响应式规则

| 弹窗类型 | 宽度规则 |
|---------|---------|
| 事项编辑弹窗 | 固定 900px，超出 80% 屏幕宽度时使用 80% |
| 材料编辑弹窗 | 固定 700px，超出 80% 屏幕宽度时使用 80% |
| 确认对话框 | 固定 400px |
| 文件预览弹窗 | 固定 800px × 600px |

## 9. 界面状态说明

### 9.1 加载状态

| 场景 | 加载方式 |
|------|---------|
| 列表加载 | 表格 loading 遮罩 + 骨架屏 |
| 详情加载 | 页面骨架屏 |
| 表单提交 | 按钮 loading 状态 + 禁用按钮 |
| 文件上传 | 进度条显示（0-100%） |

### 9.2 空状态

| 场景 | 显示内容 |
|------|---------|
| 列表无数据 | `暂无事项数据` + 空状态插图 |
| 材料列表为空 | `暂未添加材料` + `+ 添加材料` 按钮 |
| 搜索结果为空 | `未找到符合条件的事项` + 清除筛选建议 |

### 9.3 错误状态

| 场景 | 显示内容 |
|------|---------|
| 表单校验失败 | 字段下方红色错误提示 |
| 接口请求失败 | `操作失败：{错误原因}` + ElMessage.error |
| 无权限操作 | `无权限` + 跳转 403 页面或隐藏按钮 |
| 数据不存在 | `数据不存在` + 返回列表 |
